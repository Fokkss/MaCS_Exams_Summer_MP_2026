### 1. Дискретное преобразование Фурье (DFT), обратное DFT. Свёртки и умножение многочленов при помощи алгоритма FFT. Обобщения на другие кольца, NTT. Вычислительная схема FFT (преобразования бабочки).

**Определения**

- **Свёртка (конволюция)** последовательностей $a_0, \dots, a_{n-1}$ и $b_0, \dots, b_{n-1}$ — это последовательность $c_0, \dots, c_{2n-2}$, где $c_k = \sum_{t=0}^{n-1} a_t b_{k-t}$ (считая $a_i = b_i = 0$ при $i \notin [0, n-1]$).

- **Умножение многочленов** $A(x) = \sum a_i x^i$ и $B(x) = \sum b_i x^i$ даёт многочлен $C(x) = A(x)B(x)$, коэффициенты которого являются свёрткой коэффициентов $A$ и $B$.

- **Дискретное преобразование Фурье (DFT)**: для вектора $\mathbf{a} = (a_0, \dots, a_{N-1})$ и примитивного корня $N$-й степени из единицы $\omega$ (например, $\omega = e^{2\pi i / N}$) определяется как $\mathbf{y} = F_\omega \mathbf{a}$, где $y_k = \sum_{j=0}^{N-1} a_j (\omega^k)^j$.

**Основная идея FFT**

Наивное умножение многочленов требует $O(n^2)$ операций. FFT позволяет сделать это за $O(n \log n)$, используя следующий подход (интерполяция):
1. Выбрать $N \ge 2n-1$ (обычно $N = 2^t$) и дополнить коэффициенты $A$ и $B$ нулями до длины $N$.
2. Вычислить значения $A(\omega^k)$ и $B(\omega^k)$ для $k = 0, \dots, N-1$ с помощью **прямого FFT**.
3. Покомпонентно перемножить значения: $y_k = A(\omega^k) B(\omega^k)$.
4. По значениям $y_k$ восстановить коэффициенты многочлена $C$ с помощью **обратного FFT**.

**Реализация FFT (рекурсивная "бабочка")**

Идея рекурсивного разбивания ("разделяй и властвуй"): разбить $A(x)$ на чётные и нечётные коэффициенты:
- $A_0(x) = a_0 + a_2 x + \dots + a_{N-2} x^{N/2-1}$
- $A_1(x) = a_1 + a_3 x + \dots + a_{N-1} x^{N/2-1}$

Тогда $A(x) = A_0(x^2) + x A_1(x^2)$. Подставляя $x = \omega^k$, получаем:
$$A(\omega^k) = A_0(\omega^{2k}) + \omega^k A_1(\omega^{2k})$$

Так как $\omega^{2k}$ пробегают лишь $N/2$ различных значений, задача сводится к двум подзадачам размера $N/2$. Склейка результатов ("бабочка"):
- $y_k = y_k^{(0)} + \omega^k y_k^{(1)}$
- $y_{k+N/2} = y_k^{(0)} - \omega^k y_k^{(1)}$ (так как $\omega^{k+N/2} = -\omega^k$)

**Algorithm 1. Рекурсивный FFT (из `algorithms.pdf`, стр. 4)**
```
Algorithm FFT(A, ω, N)
    Input: массив коэффициентов A = [a₀, a₁, ..., a_{N-1}], где N = 2^t,
           ω — примитивный корень N-й степени из единицы
    Output: массив значений y_k = A(ω^k) для k = 0, ..., N-1

1:  if N == 1 then
2:      return [a₀]
3:  // Разбиваем на чётные и нечётные коэффициенты
4:  A₀ = [a₀, a₂, a₄, ..., a_{N-2}]      // длина N/2
5:  A₁ = [a₁, a₃, a₅, ..., a_{N-1}]      // длина N/2
6:  // Рекурсивно вычисляем FFT от половин
7:  y₀ = FFT(A₀, ω², N/2)               // значения A₀(ω^{2k})
8:  y₁ = FFT(A₁, ω², N/2)               // значения A₁(ω^{2k})
9:  // Склейка результатов ("бабочки")
10: for k = 0 to N/2 - 1 do
11:     y[k]      = y₀[k] + ω^k · y₁[k]
12:     y[k + N/2] = y₀[k] - ω^k · y₁[k]
13: return y
```

**Algorithm 2. Итеративный FFT (из `1 FFT.pdf`, стр. 4)**
```
Algorithm FFT(a, n)
    Input: массив a[0..n-1] коэффициентов многочлена, n = 2^t
    Output: массив a[0..n-1] значений в точках ω^0, ω^1, ..., ω^{n-1}
            (результат записывается на место входного массива)

1:  // Предварительное вычисление битовой перестановки
2:  for i = 0 to n-1 do
3:      rev[i] = 0
4:      for j = 0 to t-1 do
5:          if (i >> j) & 1 then
6:              rev[i] = rev[i] | (1 << (t-1-j))
7:  // Перестановка элементов в соответствии с rev
8:  for i = 0 to n-1 do
9:      if i < rev[i] then
10:         swap(a[i], a[rev[i]])
11: // Итеративное построение FFT снизу вверх
12: len = 1
13: while len < n do
14:     // ω_len = e^{2πi/(2·len)} — примитивный корень степени 2·len
15:     for i = 0 to n-1 step 2·len do
16:         // Слияние двух половин: [i .. i+len-1] и [i+len .. i+2·len-1]
17:         w = 1
18:         for j = 0 to len-1 do
19:             u = a[i + j]
20:             v = w · a[i + len + j]
21:             a[i + j]       = u + v
22:             a[i + len + j] = u - v
23:             w = w · ω_len
24:     len = len * 2
25: return a
```

**Обратное FFT**

Матрица DFT $F_\omega$ — это матрица Вандермонда. Можно показать, что $F_\omega^{-1} = \frac{1}{N} F_{\omega^{-1}}$. То есть обратное преобразование — это прямое FFT с корнем $\omega^{-1}$, после чего каждый коэффициент делится на $N$.

**Вычислительная схема FFT**

Схема FFT имеет рекурсивную структуру: два блока $FFT_{N/2}$, к выходам которых добавляется слой из $N/2$ "бабочек". Бабочка — элементарный шаг, который умножает один вход на $\omega^k$ и выполняет попарное сложение и вычитание.
- **Глубина схемы**: $O(\log N)$.
- **Размер схемы**: $O(N \log N)$.

**Обобщения (NTT)**

FFT работает в любом кольце/поле, где:
1. Существует примитивный корень $\omega$ порядка $N$.
2. $N$ обратимо в кольце (т.е. можно делить на $N$ для обратного преобразования).

- **Теоретико-числовое преобразование (NTT)**: Работает в поле $\mathbb{Z}_p$ (остатки по модулю простого числа $p$), где $p = c \cdot 2^k + 1$. Тогда существует первообразный корень $g$ по модулю $p$, и можно взять $\omega \equiv g^c \pmod{p}$, который будет иметь порядок $N = 2^k$. NTT позволяет избежать ошибок округления, присущих работе с комплексными числами.

**Предложение** $F_w^2$ - перестановачная.

**Доказательство.**

$$
(F_w ^2)_{ij} = \underset{k = 0} \sum w^{ik} w^{jk} = \underset{k = 0} \sum w^{k(i + j)} =  \\ \begin{cases} n,  \ i+j \underset {n} = 0\\ 0, \neq \end{cases}
$$



### 2. Вычисления схемами как модель параллельных вычислений. Компараторы, сортирующие сети. Принцип нулей-единиц. Битонная сортировка и сортировка чётно-нечётным слиянием.

**Вычислительная схема**

Это ациклический ориентированный граф, где:
- **Входные вершины** имеют степень входа 0.
- **Выходные вершины** имеют степень исхода 0.
- **Внутренние вершины** помечены элементарной операцией (арифметической, булевой и т.д.).
- **Размер схемы** — число внутренних вершин.
- **Глубина схемы** — длина максимального пути от входа к выходу (параллельное время вычисления).

**Сортирующие сети**

**Прикол**

1. Размер сортирующей $ = \Omega(nlog \ n)$
2. Глубина сортирующей $ \ge log(n) $

Это схема, которая сортирует последовательность чисел, используя в качестве внутренних элементов только **компараторы**. Компаратор с входами $u, v$ выдаёт $u' = \min\{u, v\}$, $v' = \max\{u, v\}$.

**Принцип нулей и единиц (Теорема 2.1 из `algorithms.pdf`)**

Сортирующая сеть корректна для всех вещественных входов **тогда и только тогда**, когда она корректна для всех входов из нулей и единиц.

**Доказательство.** Необходимость очевидна: {0,1}-входы являются частным случаем вещественных.

Докажем достаточность. Предположим, что сеть корректно сортирует все {0,1}-входы, но не сортирует некоторый вещественный вход $x = (x_1, \dots, x_n)$. Тогда на выходе получается последовательность $y = (y_1, \dots, y_n)$, для которой существуют индексы $i < j$ такие, что $y_i > y_j$.

Выберем число $t$ так, что $y_j < t \le y_i$, и определим булевый вход
$$\tilde{x}_k = \begin{cases} 0, & x_k < t, \\ 1, & x_k \ge t. \end{cases}$$

**Ключевое наблюдение.** Пороговая функция $[\cdot \ge t]$ коммутирует с каждым компаратором. Действительно, рассмотрим компаратор на входах $u, v$, вычисляющий $(\min\{u, v\}, \max\{u, v\})$:
- Если $u, v < t$, то $\min$ и $\max$ тоже $< t$; после порогования: $(0,0)$. Пороговые значения $(u, v) = (0,0)$; компаратор на них даёт $(0,0)$. Совпадает.
- Если $u, v \ge t$, то $\min$ и $\max$ тоже $\ge t$; после порогования: $(1,1)$. $(u, v) = (1,1)$; компаратор на них даёт $(1,1)$. Совпадает.
- Если $u < t \le v$ (или наоборот), то $\min\{u, v\} < t$ и $\max\{u, v\} \ge t$; после порогования: $(0,1)$. $(u, v) = (0,1)$ (или $(1,0)$); компаратор на них даёт $(0,1)$. Совпадает.

Таким образом, если сеть переводит $x$ в $y$, то она переводит $\tilde{x}$ в $\tilde{y}$, где
$$\tilde{y}_k = \begin{cases} 0, & y_k < t, \\ 1, & y_k \ge t. \end{cases}$$

Из выбора $t$ следует, что $\tilde{y}_i = 1$ и $\tilde{y}_j = 0$, то есть выход $\tilde{y}$ не отсортирован. Получено противоречие с корректностью сети на {0,1}-входах. ■

**Algorithm 3. Битонное слияние BI_MERGE (из `algorithms.pdf`, стр. 13)**
```
Algorithm BI_MERGE(z1, . . . , zm)
    Require: Битонная последовательность z1, . . ., zm, где m = 2^t для некоторого t ≥ 0
    Ensure: Последовательность z1, . . ., zm, отсортированная по неубыванию

1:  if m = 1 then
2:      return (z1)
3:  for k = 1, . . . , m/2 do
4:      z_min[k] = min(z_k, z_{k+m/2})
5:      z_max[k] = max(z_k, z_{k+m/2})
6:  L = BI_MERGE(z_min, m/2)
7:  R = BI_MERGE(z_max, m/2)
8:  return L || R        // конкатенация
```

**Лемма 2.1 (из `algorithms.pdf`).** Пусть $z_1, \dots, z_m$ — битонная последовательность. Тогда после одного слоя компараторов:
1. последовательность $(z_1^{\min}, \dots, z_m^{\min})$ битонна;
2. последовательность $(z_1^{\max}, \dots, z_m^{\max})$ битонна;
3. каждый элемент первой последовательности не превосходит каждого элемента второй.

Более точно, в случае {0,1}-последовательностей: либо $z^{\min} = (0, \dots, 0)$ и $z^{\max}$ битонна, либо $z^{\min}$ битонна и $z^{\max} = (1, \dots, 1)$.

**Лемма 2.2.** Глубина сети BI_MERGE на входе длины $m$ равна $O(\log m)$.

**Доказательство.** На каждом рекурсивном уровне добавляется один слой компараторов, после чего задача распадается на две независимые подзадачи длины $m/2$. Для глубины $T(m)$:
$$T(m) = T(m/2) + O(1),\quad T(m) = O(\log m).$$

**Лемма 2.3.** Размер сети BI_MERGE на входе длины $m$ равен $O(m \log m)$.

**Доказательство.** На верхнем уровне используется $m/2$ компараторов, после чего возникают две подзадачи длины $m/2$:
$$S(m) = 2S(m/2) + O(m),\quad S(m) = O(m \log m).$$

**Algorithm 4. Битонная сортировка BITONIC_SORT (из `algorithms.pdf`, стр. 14)**
```
Algorithm BITONIC_SORT(x1, . . . , xn)
    Require: Последовательность x1, . . . , xn, где n = 2^t для некоторого t ≥ 0
    Ensure: Последовательность из тех же элементов, отсортированная по неубыванию

1:  if n = 1 then
2:      return (x1)
3:  (a1, . . . , a_{n/2}) = BITONIC_SORT(x1, . . . , x_{n/2})
4:  (b1, . . . , b_{n/2}) = BITONIC_SORT(x_{n/2+1}, . . . , xn)
5:  return BI_MERGE(a1, . . . , a_{n/2}, b_{n/2}, . . . , b1)
```

**Теорема 2.2.** Битонная сортировка имеет глубину $O(\log^2 n)$ и размер $O(n \log^2 n)$.

**Доказательство.** Обозначим через $D(n)$ глубину схемы, а через $S(n)$ — её размер.

**Глубина.** После рекурсивной сортировки половин (глубина $D(n/2)$) выполняется BI_MERGE глубины $O(\log n)$:
$$D(n) = D(n/2) + O(\log n).$$
Раскрывая рекуррентность:
$$D(n) = \sum_{k=0}^{\log n-1} O\left(\log \frac{n}{2^k}\right) = O\left(\sum_{k=0}^{\log n-1} (\log n - k)\right) = O(\log^2 n).$$

**Размер.** Шаг BI_MERGE имеет размер $O(n \log n)$, поэтому:
$$S(n) = 2S(n/2) + O(n \log n),\quad S(n) = O(n \log^2 n).$$

**Algorithm 5. Чётно-нечётное слияние ODD_EVEN_MERGE (из `algorithms.pdf`, стр. 16)**
```
Algorithm ODD_EVEN_MERGE(a1,...,am; b1,...,bm)
    Require: Две отсортированные по неубыванию последовательности длины m, где m = 2^t
    Ensure: Последовательность длины 2m, полученная слиянием входов и отсортированная

1:  if m = 1 then
2:      u = min(a1, b1)
3:      v = max(a1, b1)
4:      return (u, v)
5:  (c1, . . . , cm) = ODD_EVEN_MERGE(a1, a3, . . . , a_{m-1}; b1, b3, . . . , b_{m-1})
6:  (d1, . . . , dm) = ODD_EVEN_MERGE(a2, a4, . . . , am; b2, b4, . . . , bm)
7:  r1 = c1
8:  for i = 1, . . . , m − 1 do
9:      r_{2i} = min(c_{i+1}, d_i)
10:     r_{2i+1} = max(c_{i+1}, d_i)
11: r_{2m} = dm
12: return (r1, . . . , r_{2m})
```

**Лемма 2.4.** Процедура ODD_EVEN_MERGE корректно сливает две отсортированные последовательности длины $m$.

**Доказательство.** По принципу нулей и единиц рассмотрим вход из {0,1}. Докажем корректность индукцией по $m$.

**База:** $m = 1$ очевидна.

**Шаг:** Пусть $m > 1$. По предположению индукции $c$ и $d$ отсортированы. Пусть $z_a$, $z_b$ — количество нулей во входах. Тогда в $c$ и $d$ их количество равно:
$$z_c = \lceil z_a/2 \rceil + \lceil z_b/2 \rceil,\quad z_d = \lfloor z_a/2 \rfloor + \lfloor z_b/2 \rfloor.$$
Так как $0 \le \lceil x/2 \rceil - \lfloor x/2 \rfloor \le 1$, разность $z_c - z_d$ может принимать значения $\{0, 1, 2\}$.

Рассмотрим последовательность $c_1, d_1, c_2, d_2, \dots, c_m, d_m$. Если $z_c - z_d \in \{0,1\}$, она уже отсортирована. Если $z_c - z_d = 2$, то $c$ содержит на два нуля больше, чем $d$. Это создаёт единственную инверсию $(d_i = 1, c_{i+1} = 0)$ на границе раздела нулей и единиц.

Финальный слой компараторов $(d_i, c_{i+1})$ для всех $i$ устраняет эту возможную инверсию, делая последовательность полностью отсортированной. ■


### 3. Префиксные суммы (scan), схема Ладнера–Фишера. Модель CRCW/EREW PRAM. Сортировка подсчётом в модели PRAM.

**Префиксные суммы (scan)**

Задача: по массиву $x_0, \dots, x_{n-1}$ вычислить массив $y_i = \sum_{j=0}^i x_j$ для ассоциативной операции (сумма, min, max и т.д.).

**Схема Ладнера–Фишера**

Идея "подъёма и спуска" (upsweep-downsweep):
1. **Свёртка (upsweep)**: Попарно складываем соседние элементы: $s_i = x_{2i} + x_{2i+1}$.
2. **Рекурсия**: Рекурсивно вычисляем префиксные суммы $p_i$ для массива $s$.
3. **Развёртка (downsweep)**: Восстанавливаем ответ: $y_{2i+1} = p_i$, $y_{2i} = p_{i-1} + x_{2i}$ (для $i \ge 1$).

**Утверждение 3.1.** Оценки глубины и размера схемы Ладнера-Фишера: при $n = 2^m$
$$T(n) = 2 + T(n/2),\quad T(1) = 0 \implies T(n) = 2\log n,$$
$$S(n) = (n-1) + S(n/2) \implies S(n) = 2n - \log n - 2.$$

**Algorithm 6. Scan (in-place, inclusive) (из `algorithms.pdf`, стр. 19)**
```
Algorithm scan_inclusive(x)
    Input: массив x[0..n−1], n = 2^m
    Output: массив x, где x[i] = x₀ + x₁ + ... + x_i

1:  // Upsweep: накапливаем суммы на концах блоков
2:  for d = 0 to log2 n − 1 do
3:      for k = 0 to n − 1 step 2^{d+1} do
4:          x[k + 2^{d+1} − 1] += x[k + 2^d − 1]
5:  // Downsweep: добавляем префикс предыдущего блока к левому подблоку
6:  for d = log2 n − 1 downto 0 do
7:      for k = 0 to n − 1 step 2^{d+1} do
8:          if k > 0 then
9:              x[k + 2^d − 1] += x[k − 1]
10: return x
```

**Модель PRAM**

Модель параллельных вычислений с общей памятью, синхронными процессорами:
- **EREW (Exclusive Read Exclusive Write)**: Одновременное чтение и запись в одну ячейку запрещены.
- **CREW (Concurrent Read Exclusive Write)**: Чтение разрешено, запись — нет.
- **CRCW (Concurrent Read Concurrent Write)**: Чтение и запись разрешены (для записи нужно правило разрешения конфликта, например, arbitrary).
- **Класс NC**: Задачи, решаемые на PRAM за $(\log n)^{O(1)}$ времени с $n^{O(1)}$ процессорами.

**Сортировка подсчётом в PRAM (из `algorithms.pdf`, стр. 21-22)**

Дан массив $a[0..n-1]$, $a[k] \in \{0, \dots, U\}$, $U = O(1)$.

1. **Локальный подсчёт**: $p$ процессоров обрабатывают блоки длины $m = n/p$, строят таблицы частот $c_i$. Время: $O(n/p)$.
2. **Сборка и scan**: Формируется массив $C$ длины $(U+1)p$, где $C[xp + i] = c_i(x)$. Вычисляется scan по $C$. Время: $O(\log p)$ (scan на длине $O(p)$, так как $U=O(1)$).
3. **Расстановка**: Каждый процессор снова проходит по своему блоку и расставляет элементы в выходном массиве согласно вычисленным позициям. Время: $O(n/p)$.

**Утверждение 3.2.** Время работы scan на $p$ процессорах составляет $O(n/p + \log p)$. В частности, при $p = \Theta(n / \log n)$ получаем время $O(\log n)$.

**Доказательство.** Разбиваем массив на блоки длины $n/p$. Каждый процессор делает локальный scan за $O(n/p)$. Затем вычисляется scan по массиву сумм блоков длины $p$ за $O(\log p)$. Затем каждый процессор прибавляет смещение к своему блоку за $O(n/p)$. Итого $O(n/p + \log p)$. ■


### 4. Трудоёмкость локальных вычислений, коммуникации и синхронизации при вычислениях в модели барьерно-синхронного параллелизма (BSP). Префиксные суммы в BSP, оптимальность.

**Модель BSP (Bulk Synchronous Parallel) (из `algorithms.pdf`, стр. 22)**

Задаётся параметрами $p$ (число процессоров), $g$ (время передачи одного слова), $L$ (время синхронизации). Обычно предполагается, что $g \ge 1$, $L \ge 1$.

1. Модель состоит из $p$ компьютеров (или процессоров), пронумерованных числами от $1$ до $p$. Каждый из них представляет собой полноценную RAM-модель со своей локальной памятью.
2. Компьютеры объединены сетью, и каждый из них может посылать сообщения размера $m$ машинных слов другому компьютеру за $g m$ единиц времени.
3. После этапа локальных вычислений все компьютеры "синхронизируются", ожидая завершения работы всех остальных компьютеров. На этом этапе доставляются все отправленные сообщения. Каждый компьютер, получающий $m$ машинных слов, тратит $g m + L$ единиц времени: $g m$ на получение сообщения и $L$ на синхронизацию.

**Супершаг**

Алгоритм в модели BSP выполняется по супершагам. Каждый супершаг состоит из трёх последовательных этапов:
1. локальные вычисления на каждом компьютере;
2. обмен сообщениями между компьютерами;
3. синхронизация, на которой все компьютеры дожидаются друг друга.

Обозначим через $t_k^{(s)}$ время локальных вычислений, выполненных $k$-м компьютером на $s$-м супершаге. Через $c_k^{(s)}$ обозначим количество машинных слов, которые $k$-й компьютер отправил или получил на этом супершаге.

Тогда время выполнения $s$-го супершага определяется самым медленным вычислением, самой большой коммуникационной нагрузкой и временем синхронизации. Поэтому оно равно
$$\max_{k=1,\dots,p} t_k^{(s)} + \max_{k=1,\dots,p} (g c_k^{(s)}) + L.$$

**Стоимость BSP-алгоритма**

Пусть алгоритм работает $\hat{S}$ супершагов. Тогда его время работы в модели BSP удобно измерять тремя компонентами:
1. вычисления: $\sum_{s=1}^{\hat{S}} \max_{k=1,\dots,p} t_k^{(s)}$;
2. коммуникация: $g \sum_{s=1}^{\hat{S}} \max_{k=1,\dots,p} c_k^{(s)}$;
3. синхронизация: $\hat{S} L$.

**Префиксные суммы в BSP (из `algorithms.pdf`, стр. 24)**

Рассмотрим задачу вычисления префиксных сумм в модели BSP. Дана последовательность из $n$ чисел $x_1, x_2, \dots, x_n$, где $n \ge p$. Требуется вычислить последовательность $y_k = \sum_{i=1}^k x_i$.

Для простоты будем считать, что $p$ делит $n$. Тогда $k$-й компьютер обрабатывает блок $x_{(k-1)n/p+1}, \dots, x_{kn/p}$.

**Первый супершаг.** $k$-й компьютер считывает из внешнего хранилища свой блок длины $n/p$. Вычисления не производятся, коммуникации: $O(n/p)$.

**Второй супершаг.** $k$-й компьютер вычисляет сумму элементов своего блока и отправляет эту сумму компьютеру 1. Вычисления: $O(n/p)$, коммуникации: $O(p)$.

**Третий супершаг.** Компьютер 1 вычисляет префиксные суммы сумм блоков, после чего отправляет $k$-му компьютеру сумму всех предыдущих блоков, которую нужно прибавить к локальным префиксным суммам внутри $k$-го блока. Вычисления: $O(p)$, коммуникации: $O(p)$.

**Четвёртый супершаг.** Каждый компьютер вычисляет префиксные суммы внутри своего блока, прибавляет к ним полученную добавку и записывает результат в выходной массив. Вычисления: $O(n/p)$, коммуникации: $O(n/p)$.

**Общее время работы:**
- вычисления: $O(p + n/p) = O(n/p)$;
- коммуникации: $O(p + n/p) = O(n/p)$;
- синхронизация: $O(1)$, так как число супершагов константно.

Если $p^2 \ll n$, то $p + n/p = O(n/p)$. Следовательно, и вычисления, и коммуникации имеют порядок $O(n/p)$.

**Оптимальность.** Алгоритм асимптотически оптимален: каждое входное машинное слово нужно хотя бы один раз прочитать, поэтому требуется не менее $\Omega(n/p)$ коммуникаций и не менее $\Omega(n/p)$ локальных вычислений.


### 5. Параллельное вычисление обобщённых линейных рекуррентных соотношений и параллельное сложение двоичных чисел при помощи задачи префиксного накопления.

**Вычисление линейной рекурренты (из `algorithms.pdf`, стр. 19)**

Дана рекуррента: $c_0 = 0$, $c_{k+1} = a_k + b_k c_k$.

Определим строчный вектор $C_k = (1, c_k)$. Тогда
$$C_{k+1} = C_k X_k, \qquad X_k = \begin{pmatrix} 1 & a_k \\ 0 & b_k \end{pmatrix}.$$

Отсюда
$$C_1 = C_0 X_0,\quad C_2 = C_0 X_0 X_1,\quad C_3 = C_0 X_0 X_1 X_2,\quad \dots$$

Таким образом, задача сводится к вычислению префиксных произведений матриц $X_0, X_1, \dots, X_{n-1}$. Поскольку умножение матриц ассоциативно, это делается через scan за $O(\log n)$ времени в параллельной модели.

**Обобщение.** В примере с матрицами при перемножении используются две операции над элементами:
$$(A \cdot B)_{ij} = \bigoplus_k (A_{ik} \odot B_{kj}).$$

Поэтому обобщение работает не для одной операции, а для полукольца $(S, \oplus, \odot)$, в котором:
- $\oplus$ ассоциативна (обычно коммутативна) и имеет нейтральный элемент 0;
- $\odot$ ассоциативна и имеет нейтральный элемент 1;
- $\odot$ дистрибутивна относительно $\oplus$: $a \odot (b \oplus c) = (a \odot b) \oplus (a \odot c)$, $(a \oplus b) \odot c = (a \odot c) \oplus (b \odot c)$;
- 0 поглощает умножение: $0 \odot a = a \odot 0 = 0$.

Примеры полуколец: $(\{0, 1\}, \lor, \land)$, $(\{0, 1\}, \oplus, \land)$, $(\mathbb{R} \cup \{+\infty\}, \min, +)$.

**Параллельное сложение двоичных чисел (Сумматор Ладнера–Фишера) (из `algorithms.pdf`, стр. 20)**

**Определение.** Сумматор — это вычислительная схема над булевыми операциями $(\lor, \land, \oplus, \lnot, \dots)$, которая принимает на вход биты $x_0, \dots, x_{n-1}$ и $y_0, \dots, y_{n-1}$ и выдаёт биты $z_0, \dots, z_n$ такие, что $z = x + y$, где
$$x = \sum_{k=0}^{n-1} x_k 2^k,\quad y = \sum_{k=0}^{n-1} y_k 2^k,\quad z = \sum_{k=0}^{n} z_k 2^k.$$

**Биты переноса и сведение к scan.** Обозначим через $c_k$ перенос в разряд $k$ (по определению $c_0 = 0$). Тогда для $k = 0, \dots, n-1$ выполняется формула переноса:
$$c_{k+1} = (x_k \land y_k) \lor (x_k \land c_k) \lor (y_k \land c_k).$$

Введём обозначения $a_k := x_k \land y_k$ и $b_k := x_k \lor y_k$. Тогда формула переноса принимает компактный вид:
$$c_{k+1} = a_k \lor (b_k \land c_k).$$

Если все переносы $c_0, \dots, c_n$ известны, ответ восстанавливается по формулам:
$$z_k = x_k \oplus y_k \oplus c_k \quad (k = 0, \dots, n-1),\qquad z_n = c_n.$$

**Ключевая идея: сведение к scan.** Рекуррента $c_{k+1} = a_k \lor (b_k \land c_k)$ является аффинным преобразованием. Положим
$$C_k = (1, c_k),\qquad X_k = \begin{pmatrix} 1 & a_k \\ 0 & b_k \end{pmatrix},$$
где операции понимаются над булевым полукольцом $(\{0,1\}, \lor, \land)$. Тогда
$$C_{k+1} = C_k X_k,$$
и задача нахождения всех переносов сводится к вычислению префиксных произведений
$$C_k = C_0 X_0 X_1 \cdots X_{k-1},$$
что выполняется через scan за $O(\log n)$ времени в параллельной модели.


### 6. Алгоритмы во внешней памяти. Сортировка слиянием. Сортировка связного списка (list ranking).

**Модель внешней памяти (из `algorithms.pdf`, стр. 25)**

Модель внешней памяти задаётся двумя параметрами: $B$ и $M$. Обычно предполагается, что $B^2 < M$. Алгоритм располагает неограниченной внешней памятью. За единицу времени он может прочитать или записать один блок, состоящий из $B$ подряд идущих машинных слов внешней памяти. Кроме того, у алгоритма есть внутренняя память объёма $M$ машинных слов, то есть $M/B$ блоков.

Все вычисления, выполняемые над данными, уже находящимися во внутренней памяти, считаются бесплатными. Входные данные изначально расположены во внешней памяти.

**Время работы** алгоритма в этой модели — число операций ввода-вывода, то есть число прочитанных и записанных блоков.

**Примеры задач в модели внешней памяти**

**Пример: Сумма N чисел.** Чтобы просуммировать N чисел, нужно хотя бы один раз прочитать весь массив. За одну операцию ввода-вывода читается блок из B элементов, поэтому время работы равно $\lceil N/B \rceil = O(N/B)$. Эту величину часто обозначают через $scan(N)$, то есть время линейного прохода по данным.

**Пример: Поиск и вставка в B-дереве.** Операции поиска и вставки выполняются за $O(\log_B N) = O(\log_B (N/B))$, поскольку один узел B-дерева помещается в один блок внешней памяти, а переход на следующий уровень дерева соответствует одной операции чтения блока.

**Сортировка слиянием во внешней памяти (из `algorithms.pdf`, стр. 25-26)**

Пусть дан массив длины $N$. Разобьём его на $N/B$ подмассивов и рекурсивно отсортируем каждый из них.

После этого из каждого подмассива читается по одному блоку. Так как число подмассивов равно $\Theta(M/B)$, все эти блоки помещаются во внутреннюю память. Поэтому слияние можно выполнять, используя только данные из внутренней памяти. Когда очередной блок какого-то подмассива заканчивается, из внешней памяти дочитывается следующий блок этого подмассива за $O(1)$. В результате на всё слияние требуется $O(N/B)$ операций ввода-вывода.

Отсюда получается рекуррентная оценка
$$T(N) = \Theta\left(\frac{M}{B}\right) T\left(\frac{N}{M/B}\right) + O\left(\frac{N}{B}\right).$$

Решение этой рекуррентной формулы даёт
$$T(N) = O\left(\frac{N}{B} \log_{M/B} \frac{N}{B}\right).$$

Эквивалентно, с точностью до константы в логарифме,
$$T(N) = O\left(\frac{N}{B} \log_{M/B} \frac{N}{M}\right).$$

Массив делим на блоки N/M берем с первых M/B блоков часть длиной B. Сливаем во внутренней памяти, пока полностью не сольем первые M/B блоков, после каждого шага количество блоков уменьшается в $M/B$ раз. и каждый раз считываем $N/B$ из внешей памяти.

$$
T(n) = \underset{i = 0} \sum^{log_{\frac{M}{B} } \frac{N}{M}} \frac{N}{B}
$$

Эту величину обозначают через $sort(N)$.

**Сортировка связного списка (List Ranking) (из `algorithms.pdf`, стр. 26-31)**

**Задача.** Дан список, хранящийся в массиве $A[0..N-1]$ в произвольном порядке. У каждого узла $x$ есть поля $x.\text{next}$, $x.w$. Также в реализации удобно хранить поле $x.id$, причём изначально $A[i].id = i$.

Исходная задача состоит в том, чтобы для каждой вершины $x$ определить её номер в списке, считая от головы: $1, 2, \dots, N$. Эквивалентно, нужно упорядочить вершины массива $A$ в порядке их следования вдоль списка.

Для решения этой задачи удобно рассмотреть более общую подзадачу: для каждой вершины $x$ вычислить сумму весов на префиксе списка до неё, то есть $pref(x) = \sum_{y \text{ от головы списка до } x} y.w$. Если изначально у всех вершин $w = 1$, то $pref(x)$ есть просто номер вершины $x$ в списке.

**Идея сжатия списка.** Пусть $S$ — множество удаляемых узлов, такое что никакие два соседних по списку узла не удаляются одновременно: $\forall x \in S$ $pred(x) \notin S$, $|S| = \Theta(N)$. Тогда все узлы из $S$ можно удалить независимо друг от друга: для каждого $x \in S$ выполняем
$$pred(x).\text{next} = x.\text{next},\qquad pred(x).w = x.w.$$

После этого каждый оставшийся узел хранит суммарный вес некоторого подряд идущего блока исходных вершин. На получившемся более коротком списке мы рекурсивно решаем ту же самую задачу. После возврата из рекурсии удалённые вершины нужно восстановить и дописать для них ответы.

**Как выбрать множество $S$.** Каждой вершине независимо присваиваем случайный бит $x.r \in \{0,1\}$, $\Pr[x.r = 0] = \Pr[x.r = 1] = 1/2$. После этого выбираем
$$x \in S \iff x.r = 1 \text{ и } pred(x).r = 0.$$

**Почему это корректно.** Если $x \in S$, то по определению $pred(x).r = 0$, $x.r = 1$. Значит, $pred(x)$ не может одновременно лежать в $S$, потому что для этого у него должен был бы быть бит $1$. Следовательно, $\forall x \in S: pred(x) \notin S$. Поэтому $\mathbb{E}|S| \approx N/4$, а после одного шага остаётся примерно $N - |S| \approx 3N/4$ вершин.

**Algorithm 7. Концептуальный алгоритм префиксных сумм для списка (из `algorithms.pdf`, стр. 28)**
```
Algorithm PrefixSums-Conceptual(A)
    Require: Текущий список А
    Ensure: Для каждой вершины x значение x.w равно сумме весов на префиксе до x

1:  if |A| ≤ 1 then
2:      return A
3:  Для каждой вершины x ∈ A независимо выбрать случайный бит x.r ∈ {0, 1}
4:  S = { x ∈ A | pred(x) существует, pred(x).r = 0, x.r = 1 }
5:  // Сжимаем список, удаляя вершины из S
6:  for all x ∈ S do
7:      p = pred(x)
8:      p.w = p.w + x.w
9:      p.next = x.next
10: A′ = A \ S
11: PrefixSums-Conceptual(A′)
12: // Восстанавливаем удалённые вершины
13: for all x ∈ S в порядке, обратном удалению do
14:     p = pred(x)
15:     p.next = x
16:     p.w = p.w − x.w        // теперь p.w = префиксная сумма до p
17:     x.w = x.w + p.w        // теперь x.w = префиксная сумма до x
18: return A
```

**Реализация через сортировки во внешней памяти.** Главная цель — заменить случайные обращения к указателям на несколько сортировок и линейных проходов. У каждого узла есть поле $x.id$, $A[i].id = i$. Это позволяет после перестановок и сортировок восстановить "исходную" вершину.

**Оценка сложности.** Обозначим через $sort(N)$ стоимость сортировки $N$ записей во внешней памяти. Один уровень рекурсии состоит из:
- константного числа сортировок по полям $id$ и $next$;
- константного числа линейных проходов по массивам.

Поэтому существует константа $c > 0$ такая, что работа одного уровня рекурсии не превосходит $c \cdot sort(N)$.

**Доказательство по индукции.** Докажем по индукции, что $\mathbb{E}[T(n)] \le 4c \cdot sort(n)$.

База индукции для малых $n$ очевидна, если увеличить константу $c$ при необходимости.

Шаг индукции:
$$\mathbb{E}[T(n)] = \sum_{k=0}^{n-1} \Pr[|S| = k] \cdot \mathbb{E}[T(n-k)] + c \cdot sort(n).$$

По предположению индукции:
$$\mathbb{E}[T(n)] \le \sum_{k=0}^{n-1} \Pr[|S| = k] \cdot 4c \cdot sort(n-k) + c \cdot sort(n).$$

Используем стандартную оценку $sort(m) = \Theta\left(\frac{m}{B} \log_{M/B} \frac{m}{B}\right)$. Тогда, поскольку $n-k \le n$,
$$sort(n-k) \le \frac{n-k}{B} \log_{M/B} \frac{n}{B}$$
с точностью до постоянного множителя, уже поглощённого константой $c$. Следовательно,
$$\mathbb{E}[T(n)] \le 4c \log_{M/B} \frac{n}{B} \sum_{k=0}^{n-1} \Pr[|S| = k] \frac{n-k}{B} + c \cdot sort(n).$$

Но $\sum_{k=0}^{n-1} \Pr[|S| = k](n-k) = \mathbb{E}[n - |S|]$. Так как каждая внутренняя вершина удаляется с вероятностью $1/4$, в среднем после сжатия остаётся не более $3n/4$ вершин, то есть $\mathbb{E}[n - |S|] \le 3n/4$.

Подставляя это в предыдущую оценку, получаем
$$\mathbb{E}[T(n)] \le 4c \log_{M/B} \frac{n}{B} \cdot \frac{3n}{4B} + c \cdot sort(n) \le 3c \cdot sort(n) + c \cdot sort(n) = 4c \cdot sort(n).$$

Индукционный переход доказан, значит $\mathbb{E}[T(n)] = O(sort(n))$.

**Итог.** Таким образом, ожидаемая сложность алгоритма равна
$$\mathbb{E}[T(N)] = O(sort(N)) = O\left(\frac{N}{B} \log_{M/B} \frac{N}{B}\right).$$


### 7. Задача линейного программирования. Допустимые и оптимальные решения. Стандартная и каноническая формы. Существование оптимального решения. Лемма Фаркаша.

**Общая постановка задачи ЛП**

Линейное программирование (ЛП) — это задача оптимизации линейной функции при линейных ограничениях.

В общем виде задача может быть записана так:
$$\max_{x \in \mathbb{R}^n} c^T x \quad \text{при} \quad Ax \le b,$$
где $A \in \mathbb{R}^{m \times n}$, $b \in \mathbb{R}^m$, $c \in \mathbb{R}^n$. Неравенство $Ax \le b$ понимается покомпонентно, то есть $(Ax)_i \le b_i$, $i = 1, \dots, m$.

**Формы записи**

Две наиболее употребительные записи:
- **Стандартная форма**: $\max c^T x$ при $Ax \le b$, $x \ge 0$.
- **Каноническая форма**: $\min c^T x$ при $Ax = b$, $x \ge 0$.

Обе формы эквивалентны общей задаче после несложных преобразований.

**Эквивалентные преобразования задач ЛП (из `algorithms.pdf`, стр. 34-35)**

1. **Минимизацию можно заменить максимизацией**: $\min c^T x \iff \max (-c)^T x$.

2. **Ограничение вида $\ge$ можно заменить ограничением вида $\le$**: если $a_k^T x \ge b_k$, то, умножив обе части на $-1$, получаем $-a_k^T x \le -b_k$.

3. **Равенство можно заменить двумя неравенствами**: если $a_k^T x = b_k$, то это эквивалентно системе $\begin{cases} a_k^T x \le b_k, \\ -a_k^T x \le -b_k. \end{cases}$

4. **Свободную переменную можно заменить разностью двух неотрицательных**: если переменная $x$ может быть любого знака, то вводят $x_j^+ \ge 0$, $x_j^- \ge 0$, и полагают $x_j = x_j^+ - x_j^-$.

5. **Неравенство $\le$ можно превратить в равенство**: если имеется ограничение $a_k^T x \le b_k$, то вводится добавочная переменная $s_k \ge 0$, такая что $a_k^T x + s_k = b_k$.

**Допустимые и оптимальные решения**

**Множество допустимых решений**: $P = \{x \in \mathbb{R}^n \mid Ax \le b\}$ называется множеством допустимых решений. Оно является выпуклым, поскольку $P$ есть пересечение полупространств, а каждое полупространство выпукло.

**Оптимальное решение**: точка $x^* \in \arg\max_{x \in P} c^T x$ называется оптимальным решением задачи линейного программирования.

**Ограниченная задача ЛП**: задача линейного программирования называется ограниченной, если $P \neq \emptyset$ и $\exists M \in \mathbb{R} \ \forall x \in P: c^T x \le M$ (в случае max; в случае min, $c^T x \ge M$).

**Вершина**: точка $x \in P$ называется вершиной, если её нельзя представить в виде нетривиальной выпуклой комбинации двух различных точек из $P$, то есть не существует $y, z \in P$, $y \neq z$, $\lambda \in (0,1)$ таких, что $x = \lambda y + (1-\lambda)z$.

**Теорема 6.1 (из `algorithms.pdf`).** Если задача линейного программирования ограничена, то её оптимум достигается.

**Лемма Фаркаша (Теорема 6.2 из `algorithms.pdf`, стр. 41)**

Пусть $A \in \mathbb{R}^{m \times n}$ и $c \in \mathbb{R}^n$. Тогда выполняется ровно одно из двух условий:
1. $\exists y \ge 0: A^T y = c$;
2. $\exists z \in \mathbb{R}^n: Az \le 0$ и $c^T z > 0$.

**Геометрическая интерпретация.** Пусть строки матрицы $A$ имеют вид $A = \begin{pmatrix} a_1^T \\ \vdots \\ a_m^T \end{pmatrix}$. Тогда множество $K = \{A^T y : y \ge 0\}$ состоит из всех линейных комбинаций вида $y_1 a_1 + \cdots + y_m a_m$, $y_1, \dots, y_m \ge 0$. Это конус, натянутый на векторы $a_1, \dots, a_m$.

Первое условие леммы Фаркаша равносильно тому, что $c \in K$. Если же $c \notin K$, то существует гиперплоскость $\{x \in \mathbb{R}^n : x^T z = 0\}$, которая отделяет $c$ от всего конуса $K$. Тогда $a_i^T z \le 0$ для всех $i = 1, \dots, m$, то есть $Az \le 0$, и одновременно $c^T z > 0$.

**Доказательство леммы Фаркаша.** Условия (1) и (2) не могут выполняться одновременно. Действительно, если $A^T y = c$, $y \ge 0$, и $Az \le 0$, то $c^T z = (A^T y)^T z = y^T Az \le 0$, что противоречит условию $c^T z > 0$.

Осталось доказать, что если (1) не выполнено, то выполнено (2). Рассмотрим конус $K = \{A^T y : y \ge 0\}$. Если (1) не выполнено, то $c \notin K$. Выберем точку $\delta \in K$, ближайшую к $c$, то есть $|c - \delta| = \inf_{u \in K} |c - u|$.

Положим $z = c - \delta$. Тогда в двумерном подпространстве $L = \operatorname{Lin}(c, \delta)$ из элементарной геометрии следует, что вектор $z$ ортогонален опорной прямой к конусу $K \cap L$ в точке $\delta$. Отсюда получаем $\delta^T z = 0$ и $c^T z > 0$.

Докажем теперь, что $Az \le 0$, то есть $a_i^T z \le 0$ для всех $i = 1, \dots, m$. Предположим противное: пусть для некоторого $k$ $a_k^T z > 0$. Так как $\delta \in K$ и $a_k \in K$, а $K$ выпукло, весь отрезок $[a_k, \delta] = \{\lambda a_k + (1-\lambda)\delta : \lambda \in [0,1]\}$ лежит в $K$. Кроме того, $\delta^T z = 0$, $a_k^T z > 0$, поэтому точки отрезка $[a_k, \delta)$ лежат по положительную сторону гиперплоскости $\{x : x^T z = 0\}$. Из геометрии в трёхмерном подпространстве $\operatorname{Lin}(c, \delta, a_k) \cong \mathbb{R}^3$ следует, что тогда на отрезке $[a_k, \delta]$ найдётся точка $\delta' \in K$, для которой $|c - \delta'| < |c - \delta|$, что противоречит выбору $\delta$ как ближайшей точки. Следовательно, $a_i^T z \le 0$ для всех $i$, то есть $Az \le 0$. Вместе с $c^T z > 0$ это и даёт (2). ■


### 8. Теорема о двойственности в линейном программировании. Системы линейных неравенств. Полиномиальные алгоритмы решения ЛП. Общая идея алгоритма внутренней точки. Преобразование задачи для нахождения внутренней точки.

**Теорема о двойственности (Теорема 6.3 из `algorithms.pdf`, стр. 43-44)**

Для $A \in \mathbb{R}^{m \times n}$, $b \in \mathbb{R}^m$, $c \in \mathbb{R}^n$ рассмотрим прямую и двойственную задачи линейного программирования:

| Прямая задача | Двойственная задача |
|---------------|---------------------|
| $\max c^T x$ | $\min b^T y$ |
| $Ax \le b$ | $A^T y = c$ |
| | $y \ge 0$ |

**Основная теорема линейного программирования.** Если одна из задач имеет оптимальное решение, то и другая имеет оптимальное решение, причём
$$c^T x^* = b^T y^*,$$
где $x^*$ и $y^*$ — оптимальные решения прямой и двойственной задач соответственно. Если целевая функция одной из задач не ограничена, то другая задача не имеет допустимых решений.

**Доказательство сильной двойственности.**

**1. Слабая двойственность.** Пусть $x$ и $y$ — допустимые решения прямой и двойственной задач. Тогда
$$c^T x = (A^T y)^T x = y^T (Ax) \le y^T b = b^T y.$$
Следовательно, $c^T x \le b^T y$ для любых допустимых $x$ и $y$. Это и есть слабая теорема двойственности.

Пусть $x^*$ — оптимальное решение прямой задачи $\max c^T x$ при $Ax \le b$. Перестановкой строк системы ограничений добьёмся того, что
$$A = \begin{pmatrix} A' \\ A'' \end{pmatrix},\quad b = \begin{pmatrix} b' \\ b'' \end{pmatrix},$$
где $A'x^* = b'$, $A''x^* < b''$. То есть в блок $A'x \le b'$ вошли активные ограничения, а в блок $A''x \le b''$ — неактивные.

Покажем, что не существует $z \in \mathbb{R}^n$ такого, что $A'z \le 0$ и $c^T z > 0$. В самом деле, если бы такой $z$ существовал, то для достаточно малого $\varepsilon > 0$ точка $x^{**} = x^* + \varepsilon z$ удовлетворяла бы $A'x^{**} = A'x^* + \varepsilon A'z \le b'$, а из строгого неравенства $A''x^* < b''$ следовало бы, что при достаточно малом $\varepsilon$ $A''x^{**} \le b''$. Значит, $Ax^{**} \le b$, то есть $x^{**}$ допустима. Но при этом $c^T x^{**} = c^T x^* + \varepsilon c^T z > c^T x^*$, что противоречит оптимальности $x^*$.

Значит, для матрицы $A'$ второе условие леммы Фаркаша не выполнено. По лемме Фаркаша тогда выполнено первое:
$$\exists y' \ge 0: (A')^T y' = c.$$

Теперь положим $y = \begin{pmatrix} y' \\ 0 \end{pmatrix}$. Тогда $y \ge 0$ и $A^T y = (A')^T y' + (A'')^T 0 = c$, то есть $y$ — допустимое решение двойственной задачи.

Проверим значения целевых функций:
$$b^T y = (b')^T y' = (A'x^*)^T y' = (x^*)^T (A')^T y' = (x^*)^T c = c^T x^*.$$

Итак, найдено допустимое $y$, для которого $b^T y = c^T x^*$. По слабой двойственности это означает, что $y$ оптимально в двойственной задаче, а $x^*$ — в прямой.

**2. Симметричность двойственности.** Осталось доказать, что если двойственная задача имеет оптимальное решение, то и прямая задача тоже имеет оптимальное решение. Преобразуем двойственную задачу к тому же виду, что и прямая:
$$\min b^T y \iff \max(-b^T y),$$
а ограничения $A^T y = c$, $y \ge 0$ перепишем в виде системы неравенств:
$$\begin{cases} A^T y \le c, \\ -A^T y \le -c, \\ -y \le 0. \end{cases}$$

Двойственная к этой задаче в точности эквивалентна исходной прямой задаче $\max c^T x$ при $Ax \le b$. Следовательно, к двойственной задаче можно применить ту же конструкцию, что и выше для прямой задачи: из оптимального решения двойственной задачи строится допустимое решение задачи, двойственной к ней, то есть прямой. Тем самым получаем оптимальное решение прямой задачи. ■

**Сведение ЛП к задаче выполнимости (из `algorithms.pdf`, стр. 46)**

Предположим, что имеется алгоритм, который за полиномиальное время решает задачу выполнимости системы линейных неравенств: либо находит решение, либо сообщает, что решений нет. Тогда задачу линейного программирования можно свести к выполнимости одной системы линейных ограничений. Действительно, $x$ и $y$ являются оптимальными решениями задач $P$ и $D$ тогда и только тогда, когда выполнена система
$$Ax \le b,\quad A^T y = c,\quad y \ge 0,\quad b^T y - c^T x \le 0.$$
По слабой двойственности для любых допустимых $x$ и $y$ всегда выполнено $b^T y - c^T x \ge 0$. Следовательно, добавленное неравенство $b^T y - c^T x \le 0$ эквивалентно равенству $b^T y = c^T x$, то есть условию оптимальности.

**Длина входа и полиномиальность (из `algorithms.pdf`, стр. 47)**

Пусть все элементы матрицы $A$, вектора $b$ и вектора $c$ являются рациональными числами, каждое из которых кодируется не более чем $w$ битами. Тогда длиной входа задачи линейного программирования называется величина
$$L = (mn + m + n)w = \Theta(mnw).$$

Алгоритм решения задачи линейного программирования называется полиномиальным, если его время работы оценивается величиной $O(L^{O(1)})$.

**Полиномиальные алгоритмы решения ЛП.** Известны следующие полиномиальные алгоритмы:
- **Метод эллипсоидов** — первый полиномиальный алгоритм, но медленный на практике.
- **Метод внутренней точки (алгоритм Кармаркара)** — полиномиальный и эффективный на практике.
- **Симплекс-метод** — работает хорошо на практике, но в худшем случае имеет экспоненциальное время. Полиномиальная гипотеза Хирша утверждает, что всегда можно за $n^{O(1)}$ перемещений по рёбрам дойти до вершины оптимума.

**Идея метода внутренней точки (из `algorithms.pdf`, стр. 47-48)**

Алгоритм движется внутри допустимого многогранника $Ax < b$ к оптимальному решению, используя штрафную ("барьерную") функцию.

**Барьерная функция.** Для $x$, лежащих во внутренности многогранника $Ax < b$, положим
$$g(x) := \sum_{i=1}^m \ln(b_i - a_i^T x) = \sum_{i=1}^m \ln s_i(x),$$
где $s_i(x) = b_i - a_i^T x$ — зазоры до ограничений.

**Барьерная задача.** Для параметра $w > 0$ рассмотрим барьерную задачу
$$\max \left(w c^T x + g(x)\right) \quad \text{при условии} \quad Ax \le b.$$
Обозначим через $x_w^*$ одно из её решений.

**Центральный путь.** Семейство точек $\{x_w^*\}_{w>0}$ образует центральный путь. Кроме того,
$$\arg\max_{Ax<b} (w c^T x + g(x)) = \arg\max_{Ax<b} \left(c^T x + \frac{1}{w} g(x)\right).$$
Поэтому при $w \to +\infty$ влияние барьерного слагаемого ослабевает, и точки $x_w^*$ приближаются к оптимальному решению исходной задачи линейного программирования.

**Геометрический смысл.** Барьерная функция $g(x)$ стремится к $-\infty$ при приближении к границе многогранника $Ax \le b$. Поэтому максимизатор $x_w^*$ остаётся во внутренности допустимой области. При увеличении $w$ эти точки движутся внутри многогранника в сторону оптимального решения исходной задачи.

**Как найти внутреннюю точку (из `algorithms.pdf`, стр. 47)**

Для поиска внутренней точки $x_0$ ($Ax_0 < b$) удобно рассмотреть вспомогательную задачу
$$\max t \quad \text{при условии} \quad Ax + t\mathbf{1} \le b,$$
где $\mathbf{1} = (1, \dots, 1)^T \in \mathbb{R}^m$. Если оптимальное значение $t^*$ положительно, то соответствующая точка $x^*$ удовлетворяет $Ax^* < b$, то есть является внутренней точкой исходной системы. У вспомогательной задачи всегда есть очевидная внутренняя точка, например $x = 0$, $t = -\max_{1 \le i \le m} |b_i| - 1$, поскольку тогда $t < b_i$ для всех $i$, а значит $Ax + t\mathbf{1} = t\mathbf{1} < b$.

**Идея метода внутренней точки (продолжение)**

Пусть $f_k(x) := w_k c^T x + g(x)$, где $0 < w_0 < w_1 < w_2 < \dots$. Идея метода состоит в том, чтобы выбирать параметры $w_k$ возрастающими и поддерживать точку $x_k$, которая достаточно близка к центральной точке $x_k^* := x_{w_k}^*$. После этого из $x_k$ выполняется один шаг Ньютона, который перемещает точку ближе к $x_{k+1}^*$.

**Шаг Ньютона (из `algorithms.pdf`, стр. 49)**

**Обозначения.** Пусть $f(x_1, \dots, x_n)$ — функция многих переменных.
- $\nabla f(x)$ обозначает градиент функции $f$ в точке $x$, то есть столбец её первых частных производных:
$$\nabla f(x) = \begin{pmatrix} \frac{\partial f}{\partial x_1}(x) \\ \vdots \\ \frac{\partial f}{\partial x_n}(x) \end{pmatrix}.$$
- $\nabla^2 f(x)$ обозначает гессиан функции $f$ в точке $x$, то есть матрицу её вторых частных производных:
$$\nabla^2 f(x) = \left( \frac{\partial^2 f}{\partial x_i \partial x_j} \right)_{i,j=1}^n.$$

**Предложение 7.4 (из `algorithms.pdf`, стр. 49).** Для функции
$$g(x) = \sum_{i=1}^m \ln(b_i - a_i^T x)$$
справедливы формулы
$$\nabla g(x) = -\sum_{i=1}^m \frac{a_i}{s_i(x)}, \qquad \nabla^2 g(x) = -\sum_{i=1}^m \frac{a_i a_i^T}{s_i(x)^2},$$
где $s_i(x) = b_i - a_i^T x$ — зазоры до ограничений.

Следовательно,
$$\nabla f_k(x) = w_k c - \sum_{i=1}^m \frac{a_i}{s_i(x)}, \qquad \nabla^2 f_k(x) = -\sum_{i=1}^m \frac{a_i a_i^T}{s_i(x)^2}.$$

**Доказательство.** Для каждого $i$ имеем $\nabla \ln(b_i - a_i^T x) = -\frac{a_i}{b_i - a_i^T x} = -\frac{a_i}{s_i(x)}$. Повторное дифференцирование даёт $\nabla^2 \ln(b_i - a_i^T x) = -\frac{a_i a_i^T}{(b_i - a_i^T x)^2} = -\frac{a_i a_i^T}{s_i(x)^2}$. Остаётся просуммировать по $i = 1, \dots, m$. ■

**Квадратическая аппроксимация (из `algorithms.pdf`, стр. 50).** Во второй окрестности точки $x_k$ функция $f_k$ аппроксимируется формулой Тейлора второго порядка:
$$\tilde{f}_k(x) = f_k(x_k) + \nabla f_k(x_k)^\top (x - x_k) + \frac{1}{2}(x - x_k)^\top \nabla^2 f_k(x_k)(x - x_k).$$

Точка максимума квадратической модели определяется из условия $\nabla \tilde{f}_k(x) = 0$, то есть из линейной системы
$$0 = \nabla f_k(x_k) + \nabla^2 f_k(x_k)(x - x_k).$$

Отсюда получается **шаг Ньютона**:
$$x_{k+1} = x_k - (\nabla^2 f_k(x_k))^{-1} \nabla f_k(x_k).$$

**Эллипсоид Дикина (из `algorithms.pdf`, стр. 50)**

**Определение.** Для точки $z$, удовлетворяющей $Az < b$, и радиуса $R > 0$ определим
$$\mathcal{E}(z, R) := \left\{ x \in \mathbb{R}^n : \sum_{i=1}^m \frac{(s_i(x) - s_i(z))^2}{s_i(z)^2} \le R^2 \right\}.$$

Так как $s_i(x) - s_i(z) = -a_i^\top (x - z)$, это равносильно записи
$$\mathcal{E}(z, R) = \left\{ x \in \mathbb{R}^n : (x - z)^\top \left( \sum_{i=1}^m \frac{a_i a_i^\top}{s_i(z)^2} \right) (x - z) \le R^2 \right\}.$$

**Назначение эллипсоида Дикина.** Эллипсоид Дикина задаёт естественную локальную геометрию барьерной функции и служит удобной метрикой для измерения близости точки $x_k$ к центральной точке $x_k^*$.

**Параметры алгоритма (из `algorithms.pdf`, стр. 51)**

Пусть
$$w_0 = 2^{-\Theta(L)}, \qquad w_{k+1} = \left(1 + \frac{1}{100\sqrt{m}}\right) w_k,$$
где $L$ — длина входа задачи ЛП.

**Algorithm. Метод внутренней точки**
```
Algorithm InteriorPoint(A, b, c)
    Input: матрица A ∈ ℝ^{m×n}, векторы b ∈ ℝ^m, c ∈ ℝ^n
    Output: оптимальное решение x*

1:  Выбрать внутреннюю точку x₀ (Ax₀ < b)
2:  w₀ = 2^{-Θ(L)}
3:  for k = 0, 1, 2, ... do
4:      f_k(x) = w_k·c^T x + g(x)          // g(x) = Σ ln(b_i - a_i^T x)
5:      // Шаг Ньютона
6:      x_{k+1} = x_k - (∇²f_k(x_k))^{-1}·∇f_k(x_k)
7:      w_{k+1} = (1 + 1/(100√m))·w_k
8:      if достигнута требуемая точность then
9:          return x_{k+1}
```

**Типовые инварианты и оценки (Утверждение 7.2 из `algorithms.pdf`, стр. 51).** В стандартном анализе метода внутренней точки доказываются следующие факты:

1. Если текущая точка удовлетворяет $x_k \in \mathcal{E}\left(x_k^*, \frac{1}{12}\right)$, то после умеренного увеличения параметра $w_k$ новая центральная точка $x_{k+1}^*$ остаётся близкой к $x_k^*$.
2. Один шаг Ньютона, выполненный из точки, достаточно близкой к $x_k^*$, переносит итерацию в точку, которая достаточно близка к $x_{k+1}^*$.
3. В частности, можно поддерживать инвариант вида $x_k \in \mathcal{E}\left(x_k^*, \frac{1}{12}\right)$ для всех $k$.
4. Для оптимального значения $x^*$ исходной задачи верна оценка разрыва:
   $$\left|c^T x_k - c^T x^*\right| = O\left(\frac{m}{w_k}\right).$$
5. Поэтому достаточно увеличить $w_k$ до величины порядка $w_k = \Theta(m \cdot 2^{\Theta(L)})$, чтобы обеспечить точность $\left|c^T x_k - c^T x^*\right| < 2^{-\Theta(L)}$.

**Количество итераций (из `algorithms.pdf`, стр. 52).** Пусть
$$w_k = \left(1 + \frac{1}{100\sqrt{m}}\right)^k w_0, \quad w_0 = 2^{-\Theta(L)}.$$

Тогда для достижения точности $2^{-\Theta(L)}$ необходимо довести $w_k$ от величины $2^{-\Theta(L)}$ до величины $\Theta(m \cdot 2^{\Theta(L)})$. Это эквивалентно условию
$$\frac{w_k}{w_0} = m \cdot 2^{\Theta(L)}.$$

Беря логарифм, получаем
$$k \ln \left(1 + \frac{1}{100\sqrt{m}}\right) = \Theta(L + \ln m).$$

Так как $\ln(1 + \varepsilon) = \Theta(\varepsilon)$ при малых $\varepsilon$, имеем
$$\ln \left(1 + \frac{1}{100\sqrt{m}}\right) = \Theta\left(\frac{1}{\sqrt{m}}\right).$$

Следовательно,
$$k = O(\sqrt{m}(L + \ln m)) = O(\sqrt{m} L).$$

Таким образом, метод внутренней точки требует $O(\sqrt{m} L)$ итераций, каждая из которых требует решения системы линейных уравнений (шаг Ньютона) и может быть реализована за полиномиальное время.

### 9. Задача о максимальном потоке в сети. Формулировка в ЛП. Дополняющие пути. Алгоритм Форда–Фалкерсона. Анализ трудоёмкости для целочисленных пропускных способностей. Теорема о максимальном потоке и минимальном разрезе.

**Определения (из `algorithms.pdf`, стр. 53)**

- **Транспортная сеть**: ориентированный граф $G = (V, E, c)$, где функция $c: E \to \mathbb{R}_{\ge 0}$ задаёт пропускные способности рёбер. Граф $G$ называется транспортной сетью, если в нём выделены две вершины: $s \in V$ (исток), $t \in V$ (сток), причём $\deg^-(s) = 0$, $\deg^+(t) = 0$.

- **Поток**: функция $f: E \to \mathbb{R}_{\ge 0}$, такая что:
    1. $0 \le f(u, v) \le c(u, v)$ для всех $(u, v) \in E$ (ограничение пропускной способности);
    2. Закон сохранения потока: для любой вершины $u \in V \setminus \{s, t\}$ выполнено $\sum_{(v,u) \in E} f(v, u) = \sum_{(u,w) \in E} f(u, w)$.

- **Величина потока**: $|f| = \sum_{(s,v) \in E} f(s, v)$. Эквивалентно, $|f| = \sum_{(u,t) \in E} f(u, t)$.

**Формулировка в ЛП (из `algorithms.pdf`, стр. 55)**

Задача о максимальном потоке допускает формулировку в виде задачи линейного программирования:
$$\max \sum_{(s,v) \in E} f(s,v)$$
при ограничениях
$$0 \le f(u,v) \le c(u,v) \quad \forall (u,v) \in E,$$
$$\sum_{(v,u) \in E} f(v,u) - \sum_{(u,w) \in E} f(u,w) = 0 \quad \forall u \in V \setminus \{s,t\}.$$

**Замечание.** Множество допустимых потоков замкнуто и ограничено в $\mathbb{R}^{|E|}$, следовательно, компактно. Поэтому непрерывная функция $(f(e))_{e \in E} \mapsto \sum_{(s,v) \in E} f(s,v)$ достигает на нём максимума по теореме Вейерштрасса.

**Дополняющие пути и остаточная сеть (из `algorithms.pdf`, стр. 56)**

**Определение.** Пусть $f$ — поток в сети $G = (V, E, c)$. Остаточной сетью называется ориентированный граф $G_f = (V, E_f, c_f)$, который строится следующим образом. Для каждого ребра $(u, v) \in E$:
1. если $c(u,v) - f(u,v) > 0$, то в $E_f$ добавляется прямое ребро $(u,v)$ остаточной пропускной способности $c_f(u,v) = c(u,v) - f(u,v)$;
2. если $f(u,v) > 0$, то в $E_f$ добавляется обратное ребро $(v, u)$ остаточной пропускной способности $c_f(v, u) = f(u, v)$.

**Определение.** Путь $P = (v_0 = s, v_1, \dots, v_k = t)$ в графе $G_f$ называется дополняющим путём для потока $f$. Его пропускной способностью называется число $\delta(P) = \min_{0 \le i < k} c_f(v_i, v_{i+1})$.

**Предложение 8.3 (из `algorithms.pdf`, стр. 57).** Пусть $P = (v_0 = s, v_1, \dots, v_k = t)$ — дополняющий путь в $G_f$, и $\delta = \delta(P)$. Тогда можно построить новый поток $f'$ величины $|f'| = |f| + \delta$.

**Доказательство.** Для каждого ребра пути действуем следующим образом:
1. если ребро $(v_i, v_{i+1})$ является прямым ребром исходной сети, то полагаем $f'(v_i, v_{i+1}) = f(v_i, v_{i+1}) + \delta$;
2. если ребро $(v_i, v_{i+1})$ является обратным, то есть в исходной сети имеется ребро $(v_{i+1}, v_i)$, то полагаем $f'(v_{i+1}, v_i) = f(v_{i+1}, v_i) - \delta$.

На всех остальных рёбрах поток не меняется. Поскольку $\delta \le c_f(v_i, v_{i+1})$ для каждого ребра пути, ограничения пропускной способности сохраняются. Закон сохранения потока также сохраняется: в каждой промежуточной вершине изменение входящего потока компенсируется изменением исходящего потока на ту же величину $\delta$.

Остаётся проверить, что величина потока увеличилась на $\delta$. По определению величины потока $|f| = \sum_{(s,v) \in E} f(s,v)$. Первое ребро дополняющего пути, выходящее из $s$, является прямым ребром исходной сети, поэтому на этом ребре поток увеличивается на $\delta$. Другие изменения, связанные с вершиной $s$, отсутствуют. Следовательно,
$$|f'| = \sum_{(s,v) \in E} f'(s,v) = \sum_{(s,v) \in E} f(s,v) + \delta = |f| + \delta.$$

**Предложение 8.4.** Для разреза S-T : $c(S, T) \ge |f|$.

**Доказательство.** $\triangleright$

Добавим $t'$ следуюшим образом:

$$
G' = G + t'
$$

$$
\forall (u, v) \in E : u \in S, \ v \in T \Rightarrow (u, t'), (t', v) \in E_{G'} 
$$

распределение никак не поменяется.

$$
|f| = |f|_ {G' \backslash T} \le \sum{c(u, t')} = c(S, T)
$$



**Algorithm 8. Алгоритм Форда–Фалкерсона (из `algorithms.pdf`, стр. 57 и `Лекция 8.txt`)**
```
Algorithm Ford-Fulkerson(G, s, t)
    Input: транспортная сеть G = (V, E, c), исток s, сток t
    Output: максимальный поток f

1:  f = 0
2:  while существует дополняющий путь P из s в t в остаточной сети G_f do
3:      δ = min_{e∈P} c_f(e)
4:      for каждого ребра (u, v) в P do
5:          if (u, v) — прямое ребро then
6:              f(u, v) = f(u, v) + δ
7:          else // (u, v) — обратное ребро, соответствующее (v, u) в исходной сети
8:              f(v, u) = f(v, u) - δ
9:      обновить остаточную сеть G_f
10: return f
```

**Реализация на псевдокоде (из `Лекция 8.txt`):**
```
function add_edge(u, v, c):
    adj[u].add(|E_f|); c_f.add(c); E_f.add(v)
    adj[v].add(|E_f|); c_f.add(0); E_f.add(u)

function bfs():
    for v in V: pred[v] = null
    q = queue{s}
    δ[s] = ∞
    while q ≠ empty:
        u = q.pop()
        for e in adj[u]:
            v = E_f[e]
            if c_f[e] > 0 and pred[v] == null:
                pred[v] = (u, e)
                δ[v] = min(δ[u], c_f[e])
                if v == t: return true
                q.push(v)
    return false

// основной цикл алгоритма
while bfs():
    for v = t; v ≠ s; v = u:
        (u, e) = pred[v]
        c_f[e] -= δ[t]
        c_f[e ^ 1] += δ[t]
```

**Оценки времени работы (из `algorithms.pdf`, стр. 60)**

**Предложение 8.6.** Одна итерация алгоритма Форда–Фалкерсона работает за время $O(|E|)$, если поиск дополняющего пути выполняется обходом графа в глубину или в ширину.

**Предложение 8.7.** Если все пропускные способности целочисленны, то алгоритм Форда–Фалкерсона завершится не более чем за $|f_{\max}|$ итераций, где $f_{\max}$ — максимальный поток.

**Доказательство.** При целочисленных пропускных способностях каждая остаточная пропускная способность также целочисленна. Следовательно, на каждой итерации $\delta(P) \ge 1$, и величина потока увеличивается хотя бы на единицу. ■

**Следствие.** В случае целочисленных пропускных способностей время работы алгоритма Форда–Фалкерсона оценивается как $O(|E| \cdot |f_{\max}|)$.

**Теорема о максимальном потоке и минимальном разрезе (Теорема 8.1 из `algorithms.pdf`, стр. 59)**

Для потока $f$ в транспортной сети эквивалентны следующие условия:
1. $f$ является максимальным потоком;
2. в остаточной сети $G_f$ нет дополняющего пути из $s$ в $t$;
3. существует $s$-$t$-разрез $(S, T)$ такой, что $|f| = c(S, T)$.

**Доказательство.** Докажем циклически $1) \Rightarrow 2) \Rightarrow 3) \Rightarrow 1)$.

$1) \Rightarrow 2)$. Если бы в $G_f$ существовал дополняющий путь, то по предыдущему предложению поток можно было бы увеличить. Это противоречит максимальности $f$.

$2) \Rightarrow 3)$. Пусть $S$ — множество вершин, достижимых из $s$ в остаточной сети $G_f$, а $T = V \setminus S$. Тогда $s \in S$, а $t \notin S$, поскольку по предположению пути из $s$ в $t$ в $G_f$ нет.

Покажем, что $|f| = c(S, T)$. Из доказательства оценки для разреза имеем
$$|f| = \sum_{\substack{(u,v) \in E \\ u \in S, v \in T}} f(u, v) - \sum_{\substack{(u,v) \in E \\ u \in T, v \in S}} f(u, v).$$

Если $(u, v) \in E$, $u \in S$, $v \in T$, то в остаточной сети не может быть прямого ребра $(u, v)$, иначе вершина $v$ была бы достижима из $s$. Значит, $c(u, v) - f(u, v) = 0$, то есть $f(u, v) = c(u, v)$.

Если $(u, v) \in E$, $u \in T$, $v \in S$, то в остаточной сети не может быть обратного ребра $(v, u)$, иначе вершина $u$ была бы достижима из $s$. Значит, $f(u, v) = 0$.

Следовательно,
$$|f| = \sum_{\substack{(u,v) \in E \\ u \in S, v \in T}} c(u, v) = c(S, T).$$

$3) \Rightarrow 1)$. Для любого потока $f'$ и любого разреза $(S, T)$ верно $|f'| \le c(S, T)$. Если для некоторого потока $f$ выполнено равенство $|f| = c(S, T)$, то никакой поток не может иметь величину больше $|f|$. Следовательно, $f$ максимален. ■

**Следствие.** Максимальная величина потока равна минимальной величине $s$-$t$-разреза.


### 10. Приближённые алгоритмы. Задача о вершинном покрытии минимальной мощности, приближённый полиномиальный алгоритм. Задача о вершинном покрытии минимальной стоимости, приближённое решение при помощи ЛП.

**Приближённый алгоритм (из `algorithms.pdf`, стр. 61)**

Пусть дана задача оптимизации $f: \text{input} \to \mathbb{R}$, решаемая на максимум или минимум. Для $\rho \ge 1$ алгоритм $A$ называется $\rho$-приближением, если для любого входа $x$:
- для задачи максимизации: $A(x) \ge f^*(x)/\rho$;
- для задачи минимизации: $A(x) \le f^*(x) \cdot \rho$,
где $f^*(x)$ — оптимальное значение на входе $x$.

**Задача о вершинном покрытии минимальной мощности (из `algorithms.pdf`, стр. 62)**

**Определение.** Дан граф $G = (V, E)$. Вершинное покрытие — это подмножество $C \subseteq V$ такое, что для каждого ребра $e \in E$ хотя бы один из его концов принадлежит $C$. Задача: найти вершинное покрытие минимального размера $|C|$.

**Algorithm 9. 2-приближённый алгоритм для вершинного покрытия (жадный по паросочетанию)**
```
Algorithm VertexCover(G)
    Input: граф G = (V, E)
    Output: вершинное покрытие C

1:  C = ∅
2:  while E ≠ ∅ do
3:      выбрать произвольное ребро {u, v} ∈ E
4:      C = C ∪ {u, v}
5:      удалить из E все рёбра, инцидентные u или v
6:  return C
```

**Утверждение 9.1.** Алгоритм жадного паросочетания даёт 2-приближение. Пусть $C^*$ — минимальное вершинное покрытие. Тогда $|C| \le 2|C^*|$.

**Доказательство.** В ходе алгоритма строится набор рёбер $M$ (паросочетание): на каждом шаге выбирается ребро $\{u, v\}$, причём после удаления $u$ и $v$ ни одно из последующих рёбер не касается $u$ или $v$. Таким образом, $M$ — паросочетание размера $|C|/2$.

Так как $C^*$ является вершинным покрытием, каждое ребро из $M$ должно иметь хотя бы один конец в $C^*$. Поскольку рёбра паросочетания не делят вершин, различным рёбрам из $M$ соответствуют различные вершины из $C^*$. Следовательно,
$$|C^*| \ge |M| = \frac{|C|}{2} \implies |C| \le 2|C^*|.$$

**Доказательство 2** $\triangleright$

Пусть $C_G$ — минимальное вершинное покрытие графа $G$.

$$V_G = C_G \cup V_G / C_G$$

При этом, 

$$E \mid _{V_G / C_G} = \emptyset$$

$\Rightarrow$ Алгоритм всегда берёт минимум 1 вершину из $C_G$

$\Rightarrow$ Алгоритм должен взять ВСЕ вершины из $C_G$

$$|ALG| \le 2 \cdot |OPT|$$

**Ровно 2**

Граф из $2m$ вершин, состоящий из $m$ компонентов связности с $m$ рёбрами.

$|OPT| = m$

$|ALG| = 2m$

**Задача о взвешенном вершинном покрытии (из `algorithms.pdf`, стр. 62-63)**

Дан граф $G = (V, E)$ и функция весов $w: V \to \mathbb{R}_{>0}$. Требуется найти вершинное покрытие $C \subseteq V$ минимального суммарного веса $\sum_{v \in C} w(v) \to \min$.

**Целочисленное программирование.** Введём переменные $x_v \in \{0, 1\}$ для каждой вершины $v$: $x_v = 1$ означает, что вершина $v$ включена в покрытие. Задача записывается как:
$$\begin{cases} \min \sum_{v \in V} w(v) x_v, \\ x_u + x_v \ge 1 \quad \forall \{u, v\} \in E, \\ 0 \le x_v \le 1 \quad \forall v \in V, \\ x_v \in \{0, 1\}. \end{cases}$$

**Algorithm 10. 2-приближённый алгоритм через LP-релаксацию**
```
Algorithm WeightedVertexCover(G, w)
    Input: граф G = (V, E), веса вершин w(v) > 0
    Output: вершинное покрытие C

1:  // Решаем LP-релаксацию (без ограничения x_v ∈ {0,1})
2:  min ∑ w(v) x_v
3:  при условиях: x_u + x_v ≥ 1 для всех {u,v} ∈ E
4:                0 ≤ x_v ≤ 1 для всех v ∈ V
5:  // Получаем оптимальное дробное решение {x_v^*}
6:  C = { v ∈ V | x_v^* ≥ 1/2 }
7:  return C
```

**Теорема 9.1 (о 2-приближении через LP-релаксацию).** Множество $C$, построенное описанным методом, является вершинным покрытием и 2-приближением задачи о взвешенном вершинном покрытии.

**Доказательство.**

**Корректность покрытия.** Для каждого ребра $\{u, v\} \in E$ из ограничения LP: $x_u^* + x_v^* \ge 1$. Следовательно, хотя бы одно из двух неравенств $x_u^* \ge 1/2$ или $x_v^* \ge 1/2$ выполнено, то есть хотя бы одна из вершин попадает в $C$. Значит, $C$ — вершинное покрытие.

**Оценка качества.** Так как $x_v = 1$ только при $x_v^* \ge 1/2$, имеем $x_v \le 2x_v^*$ для всех $v$. Поэтому:
$$\sum_{v \in C} w(v) \le \sum_{v \in V} w(v) \cdot 2x_v^* = 2 \sum_{v \in V} w(v) x_v^* \le 2 \sum_{v \in V} w(v) \hat{x}_v = 2 \cdot OPT,$$
где $\{\hat{x}_v\}$ — оптимальное целочисленное решение (оно допустимо для LP, поэтому $\sum w(v) x_v^* \le \sum w(v) \hat{x}_v$). ■


### 11. Метрическая задача коммивояжёра. Алгоритм двойного дерева и алгоритм Кристофидеса–Сердюкова.

**Метрическая задача коммивояжёра (из `algorithms.pdf`, стр. 64)**

**Определение.** Дан полный граф $K_n = G(V, E)$ с весами $c: E \to \mathbb{R}_{\ge 0}$ на рёбрах. Требуется найти гамильтонов цикл $H$ (цикл, проходящий через каждую вершину ровно один раз) минимального веса $\sum_{e \in E(H)} c(e)$.

TSP называется **метрическим**, если веса рёбер удовлетворяют неравенству треугольника:
$$\forall u, v, w \in V: \quad c(u, v) + c(v, w) \ge c(u, w).$$

В общем случае TSP неаппроксимируема, но для метрической версии существуют алгоритмы с константной аппроксимацией.

**Algorithm 11. Алгоритм двойного дерева (2-приближение) (из `algorithms.pdf`, стр. 65)**
```
Algorithm DoubleTreeTSP(G)
    Input: полный граф G с метрическими весами
    Output: гамильтонов цикл H

1:  T = минимальное остовное дерево (MST) графа G
```

```
Algorithm ChristofidesTSP(G)
    Input: полный граф G с метрическими весами
    Output: гамильтонов цикл H

6:  Построить гамильтонов цикл H, срезая повторные вершины
```

**Теорема 9.3.** Алгоритм Кристофидеса–Сердюкова даёт 3/2-приближение, то есть $c(H) \le \frac{3}{2} c(H^*)$.

**Доказательство.** Вес $c(H)$ не превышает суммарного веса Эйлерова пути в $T'$:
$$c(H) \le c(T) + c(M).$$

**Шаг 1:** $c(T) \le c(H^*)$. Доказано выше (в любом остовном дереве).

**Шаг 2:** $c(M) \le \frac{c(H^*)}{2}$. Пронумеруем вершины $V_0 = \{v_1, v_2, \dots, v_{|V_0|}\}$ в порядке их встречаемости при обходе оптимального гамильтонова цикла $H^*$. Рассмотрим два паросочетания на $V_0$:
$$M_1 = \{v_1v_2, v_3v_4, \dots\}, \qquad M_2 = \{v_2v_3, v_4v_5, \dots\}.$$
По неравенству треугольника (применяя обобщённое неравенство к путям $H^*$ между соответствующими вершинами $V_0$):
$$c(M_1) + c(M_2) \le c(H^*).$$
Следовательно, $\min(c(M_1), c(M_2)) \le \frac{c(H^*)}{2}$. Минимальное паросочетание на $V_0$ не хуже наилучшего из $M_1$, $M_2$:
$$c(M) \le \min(c(M_1), c(M_2)) \le \frac{c(H^*)}{2}.$$

**Итог:**
$$c(H) \le c(T) + c(M) \le c(H^*) + \frac{c(H^*)}{2} = \frac{3}{2} c(H^*).$$

**Вероятностные алгоритмы (из `algorithms.pdf`, стр. 68)**

- **Алгоритм Лас-Вегас**: всегда возвращает корректный ответ, но время работы $t(n)$ является случайной величиной.
- **Алгоритм Монте-Карло**: может вернуть некорректное решение, однако вероятность ошибки мала и поддаётся оценке.

**Определения**

$$
T(n) = \max_{A \in I_n} \mathbb{E}[\text{time}(\mathcal{A})]
$$

**Algorithm 13. Алгоритм Фрейвальдса (проверка AB = C) (из `algorithms.pdf`, стр. 69)**
```
Algorithm Freivalds(A, B, C)
    Input: матрицы A, B, C ∈ ℝ^{n×n}
    Output: «да», если AB = C, иначе «нет» (с вероятностью ошибки ≤ 1/2)

1:  Выбрать δ ∈ {0,1}^n равновероятно и покоординатно независимо
2:  Вычислить ABδ и Cδ
3:  if ABδ = Cδ then return «да»
4:  else return «нет»
```

**Теорема 10.1 (оценка ошибки алгоритма Фрейвальдса).** Если $
AB = C
$, алгоритм всегда отвечает верно. Если $AB \ne C$, то $\Pr(\text{ошибка}) \le 1/2$.

**Доказательство.** Случай $AB = C$ очевиден. Пусть $AB \ne C$. Пусть матрицы отличаются в строке $k$; обозначим $x := (AB)[k, \cdot]$, $y := C[k, \cdot]$, тогда $x \ne y$. Ошибка — событие $\{x\delta = y\delta\}$, т.е. $(x - y)\delta = 0$.

Пусть $\ell$ — индекс, в котором $x_\ell \ne y_\ell$. Зафиксируем все $\delta_i$ при $i \ne \ell$. Условие $(x-y)\delta = 0$ записывается как
$$(x_\ell - y_\ell)\delta_\ell = -\sum_{i \ne \ell} (x_i - y_i)\delta_i.$$

1. Если правая часть равна 0: равенство выполняется только при $\delta_\ell = 0$, поэтому $\Pr(x\delta = y\delta \mid \delta_{-\ell}) = 1/2$.
2. Если правая часть $\ne 0$: уравнение имеет не более одного решения $\delta_\ell \in \{0,1\}$, поэтому $\Pr(x\delta = y\delta \mid \delta_{-\ell}) \le 1/2$.

В обоих случаях $\Pr(\text{ошибка}) \le 1/2$.

**Доказательство 2** $\triangleright$

Матрица ошибки $E = AB - C$

Алгебра, 

$$
P(Ev = 0 \mid E \neq 0) = P(v \in \mathrm{ker}E ) \le P(v \in \mathrm{ker} E \mid \mathrm{dim} \ \mathrm{ker} E = n - 1)
$$

$$
|\mathrm{ker} E| = 2^{n-1}
$$

$$
|\mathbb{F}_p^n| = 2^n
$$

$\Rightarrow $

$$
P(Ev = 0) \le \frac{|\mathrm{ker} \ E|}{|\mathbb{F}_p^n|} = \frac{1}{2}
$$

**Уменьшение вероятности ошибки.** Чтобы добиться $\Pr(\text{ошибка}) < \varepsilon$, достаточно запустить алгоритм $k$ раз независимо, где $k \in \mathbb{N}$ удовлетворяет $1/2^k < \varepsilon$. Если хотя бы в одном запуске ответ «нет» — вернуть «нет»; если во всех — «да» — вернуть «да». Тогда $\Pr(\text{ошибка}) \le 1/2^k < \varepsilon$.

**Задача проверки равенства многочленов (PIT) (из `algorithms.pdf`, стр. 70)**

Даны многочлены $p(x_1, \dots, x_n)$ и $q(x_1, \dots, x_n)$ над полем $F$. Проверить: $p \equiv q$, то есть $\forall x_1, \dots, x_n: p = q$. Эквивалентно: $f := p - q \equiv 0$.

**Лемма Шварца–Зиппеля (Лемма 10.1 из `algorithms.pdf`, стр. 70).** Пусть $f(x_1, \dots, x_n) \not\equiv 0$ — многочлен над полем $F$, $\deg f = d$. Пусть $S \subseteq F$ — конечное подмножество, $0 < |S| < +\infty$. Если $x_1, \dots, x_n$ выбраны равновероятно и независимо из $S$, то
$$\Pr_{x_1, \dots, x_n \in S} (f(x_1, \dots, x_n) = 0) \le \frac{d}{|S|}.$$

**Доказательство.** Индукция по $n$.

**База $n = 1$.** Многочлен $f(x_1)$ имеет не более $\deg f$ корней, поэтому $\Pr_{x_1 \in S} (f(x_1) = 0) \le \frac{\deg f}{|S|}$.

**Шаг $n > 1$.** Запишем $f(x_1, \dots, x_n) = \sum_{k=0}^{d} p_k(x_1, \dots, x_{n-1}) x_n^k$, $d = \deg f$.

Так как $f \not\equiv 0$, существует наибольшее $k'$ такое, что $p_{k'} \not\equiv 0$.

По индукционному предположению, применённому к многочлену $p_{k'}$, имеем
$$\Pr(p_{k'}(x_1, \dots, x_{n-1}) \neq 0) \ge 1 - \frac{\deg p_{k'}}{|S|}.$$

Зафиксируем $x_1, \dots, x_{n-1}$ так, что $p_{k'}(x_1, \dots, x_{n-1}) \neq 0$. Тогда $g(x_n) := f(x_1, \dots, x_{n-1}, x_n) = \sum_{k=0}^{d} p_k(x_1, \dots, x_{n-1}) x_n^k$ является ненулевым многочленом от $x_n$ степени $k'$. Кроме того, $k' + \deg p_{k'} \le d$, поэтому $k' \le d - \deg p_{k'}$.

По базе индукции получаем
$$\Pr(g(x_n) = 0) \le \frac{k'}{|S|} \le \frac{d - \deg p_{k'}}{|S|},$$
то есть
$$\Pr(g(x_n) \neq 0) \ge 1 - \frac{d - \deg p_{k'}}{|S|}.$$

Следовательно,
$$\begin{aligned}
\Pr(f(x_1, \dots, x_n) \neq 0) &\ge \Pr(p_{k'}(x_1, \dots, x_{n-1}) \neq 0) \cdot \Pr(g(x_n) \neq 0 \mid p_{k'}(x_1, \dots, x_{n-1}) \neq 0) \\
&\ge \left(1 - \frac{\deg p_{k'}}{|S|}\right) \left(1 - \frac{d - \deg p_{k'}}{|S|}\right) \\
&= 1 - \frac{d}{|S|} + \frac{\deg p_{k'}(d - \deg p_{k'})}{|S|^2} \\
&\ge 1 - \frac{d}{|S|}.
\end{aligned}$$

Значит, $\Pr(f(x_1, \dots, x_n) = 0) \le \frac{d}{|S|}$. ■

**Алгоритм проверки (Монте-Карло).** Выбрать случайную точку из $S^n$ (например, $S = \{1, \dots, 2d\}$) и вычислить $f(r_1, \dots, r_n)$. Если $f \ne 0$, то $f \not\equiv 0$ (многочлены не равны). Если $f = 0$, то с вероятностью $\ge 1 - d/|S|$ многочлен тождественно равен нулю. Удобно считать поле бесконечным — тогда вероятность ошибки стремится к нулю с ростом $|S|$.


### 13. Хеш-таблицы, ключи. Амортизированное и ожидаемое время работы. Время работы хеш-таблицы с открытой адресацией при идеализированном случайном равновероятном хешировании.

**Хеш-таблица с открытой адресацией (из `algorithms.pdf`, стр. 72-73)**

Хранится таблица $T[0..m-1]$ такая, что $n < m$, где $n$ — количество ключей, хранящихся в таблице в данный момент. Зафиксируем функцию $h: U \to \{0, \dots, m-1\}$, которая задаёт ячейки, в которые помещаются ключи: $x \in U$ помещается в $T[h(x)]$. Функция $h$ называется хэш-функцией.

**Коллизия** — это ситуация, когда $h(x) = h(y)$ при $x \ne y$.

**Утверждение 11.1.** Если $|U| > m$, то коллизии неизбежны при любой хэш-функции $h$.

**Разрешение коллизий с помощью линейного поиска.** При линейном зондировании ячейки таблицы просматриваются последовательно, начиная с позиции $h(x)$, по кругу. В таблице используется специальный маркер $\dagger$ (tombstone) для обозначения удалённых ячеек, а символ $\emptyset$ — для обозначения пустой ячейки.

**Algorithm 14. Операции с хеш-таблицей (открытая адресация) (из `algorithms.pdf`, стр. 73 и `Лекция 11.txt`)**
```
// Константы
∅  = null          // ячейка никогда не была занята
†  = DELETED       // ячейка была занята, но теперь удалена

// Глобальные переменные
T[0..m-1]          // массив таблицы
m                  // размер таблицы
n                  // количество реальных элементов (не считая †)

// ------------------------------------------------------------
// ВСТАВКА
// ------------------------------------------------------------
Algorithm INSERT(x, v)
1:  if n * 2 > m then
2:      REHASH()                    // расширяем, если заполнено > 50%
3:  
4:  k = h(x)                        // начальный хеш-индекс
5:  firstDeleted = -1               // запоминаем первую удаленную ячейку
6:  i = 0
7:  
8:  while i < m do
9:      index = (k + i) mod m
10:     
11:     if T[index] = ∅ then
12:         if firstDeleted ≠ -1 then
13:             index = firstDeleted // вставляем в удаленную ячейку
14:         break                    // выходим из цикла
15:     
16:     if T[index] = † then
17:         if firstDeleted = -1 then
18:             firstDeleted = index // запоминаем первую удаленную
19:         i = i + 1
20:         continue                 // продолжаем искать дальше
21:     
22:     if T[index].key = x then
23:         T[index].value = v       // обновляем существующий ключ
24:         return
25:     
26:     i = i + 1                    // занято другим ключом - идем дальше
27:  end while
28:  
29:  if i = m then
30:     ERROR: "Таблица переполнена" // должно быть невозможно из-за шага 1-2
31:  
32:  T[index].key = x                // вставка в найденную ячейку
33:  T[index].value = v
34:  n = n + 1

// ------------------------------------------------------------
// ПОИСК (вспомогательная функция)
// ------------------------------------------------------------
Algorithm FIND(x)
1:  k = h(x)
2:  i = 0
3:  
4:  while i < m do
5:      index = (k + i) mod m
6:      
7:      if T[index] = ∅ then
8:          return null              // не найдено - дальше искать нельзя
9:      
10:     if T[index] ≠ † and T[index].key = x then
11:         return T[index].value    // найдено!
12:     
13:     i = i + 1                    // идем дальше (через † или другие ключи)
14:  end while
15:  
16:  return null                      // не найдено (прошли всю таблицу)

// ------------------------------------------------------------
// УДАЛЕНИЕ
// ------------------------------------------------------------
Algorithm DELETE(x)
1:  k = h(x)
2:  i = 0
3:  
4:  while i < m do
5:      index = (k + i) mod m
6:      
7:      if T[index] = ∅ then
8:          return false             // элемента нет в таблице
9:      
10:     if T[index] ≠ † and T[index].key = x then
11:         T[index] = †             // помечаем как удаленный
12:         n = n - 1
13:         return true              // успешно удален
14:     
15:     i = i + 1
16:  end while
17:  
18:  return false                     // не найден

// ------------------------------------------------------------
// РЕХЕШИРОВАНИЕ (расширение таблицы)
// ------------------------------------------------------------
Algorithm REHASH()
1:  T_old = T                       // сохраняем старую таблицу
2:  m_old = m
3:  m = 2 * m                       // удваиваем размер
4:  T = new array of size m         // создаем новую пустую таблицу
5:  for i = 0 to m-1 do
6:      T[i] = ∅
7:  n = 0                           // сбрасываем счетчик
8:  
9:  for i = 0 to m_old - 1 do
10:     if T_old[i] ≠ ∅ and T_old[i] ≠ † then
11:         INSERT(T_old[i].key, T_old[i].value)  // рекурсивная вставка
12:     
13: delete T_old                     // освобождаем память
```

**Идеализированное предположение (из `algorithms.pdf`, стр. 73).** Хэш-функция $h$ генерируется случайно и равновероятно из множества всех функций $U \to \{0, \dots, m-1\}$, то есть для каждого $x \in U$ значение $h(x)$ выбирается независимо и равновероятно из $\{0, \dots, m-1\}$.

**Теорема 11.1 (из `algorithms.pdf`, стр. 73).** При идеализированном предположении ожидаемое время работы каждой операции равно $O(1)$ (если не происходит перестройка таблицы).

**Доказательство.** Будем считать, что все ключи различны и $m = 2n$.

Время работы операции пропорционально длине «хвоста» серии, начиная с позиции $h(x)$. Обозначим:
- серия — максимальная цепочка подряд занятых ячеек $t', t'+1, \dots, t'+k-1$, где $t'-1$ и $t'+k$ пусты (нумерация по модулю $m$);
- $q_k$ — вероятность того, что в таблице существует серия длины $k$, начинающаяся в фиксированной позиции.

**Ожидаемое время.** Пусть $p_\ell$ — вероятность встретить в позиции $t = h(x)$ «хвост» длины $\ell$ некоторой серии длины $k \ge \ell$. Если серия имеет длину $k \ge \ell$, то она должна начинаться в позиции $t - (k - \ell)$, поэтому
$$p_\ell = \sum_{k=\ell}^{n} q_k.$$

Тогда ожидаемое время равно
$$O\left(\sum_{\ell=0}^{n} p_\ell \cdot \ell\right).$$

Преобразуем:
$$\sum_{\ell=0}^{n} p_\ell \cdot \ell = \sum_{\ell=0}^{n} \sum_{k=\ell}^{n} q_k \cdot \ell = \sum_{k=0}^{n} q_k \cdot (0 + 1 + 2 + \cdots + k) = \sum_{k=0}^{n} q_k \cdot \Theta(k^2).$$

**Оценка $q_k$.** Докажем, что $q_k \le e^{-\Omega(k)}$. Из этого будет следовать:
$$\sum_{k=0}^{n} q_k \cdot \Theta(k^2) \le \Theta\left(\sum_{k=0}^{n} \frac{k^2}{e^{\Omega(k)}}\right) = O(1).$$

Если в таблице есть серия в позициях $t', t'+1, \dots, t'+k-1$, значит, $k$ ключей отображены в ячейки $\{t', t'+1, \dots, t'+k-1\}$, а остальные $n-k$ ключей — вне этих ячеек.

Следовательно, так как $m = 2n$:
$$q_k \le \binom{n}{k} \left(\frac{k}{2n}\right)^k \left(1 - \frac{k}{2n}\right)^{n-k}.$$

Оценим сверху каждый множитель. Используем неравенства
$$\binom{n}{k} \le \frac{n^n}{k^k (n-k)^{n-k}}, \quad 1+x \le e^x.$$

Тогда
$$\begin{aligned}
q_k &\le \frac{n^n}{k^k (n-k)^{n-k}} \cdot \frac{k^k}{2^k n^k} \cdot \left(\frac{2n-k}{2n}\right)^{n-k} \\
&= \frac{1}{2^k} \cdot \left(\frac{2n-k}{2n-2k}\right)^{n-k} \\
&= \frac{1}{2^k} \cdot \left(1 + \frac{k}{2n-2k}\right)^{n-k} \\
&\le \frac{1}{2^k} \cdot e^{\frac{k(n-k)}{2n-2k}} \\
&= \frac{1}{2^k} \cdot e^{k/2} = \left(\frac{\sqrt{e}}{2}\right)^k = e^{-\Omega(k)}.
\end{aligned}$$

Таким образом, ожидаемое время работы равно $O(1)$. ■

**Амортизированное время работы.** Пусть дана структура данных $D$ с операциями $op_1, op_2, \dots, op_k$. Говорят, что амортизированное время работы этих операций равно $O(f(n))$, если любая последовательность из $n$ таких операций, применённых на изначально пустой структуре данных $D$, выполняется за время $O(n f(n))$.

**Утверждение (из `Лекция 11.txt`).** Ожидаемое амортизированное время работы операций в хеш-таблице равно $O(1)$. Из-за линейности математического ожидания достаточно показать, что суммарное количество операций равно $O(n)$.


### 14. Универсальные семейства хеш-функций. 2-универсальность семейства multiply-shift. Хеш-таблицы с разрешением коллизий цепочками.

**Универсальные семейства хеш-функций (из `okhotin_algorithms3_2025_l5.pdf`, стр. 2)**

Семейство $H$ хэш-функций из $U$ в $\{0, \dots, m-1\}$ называется **универсальным**, если для всяких двух различных $x, y \in U$ вероятность того, что для случайно выбранной функции $h \in H$ значения $h(x)$ и $h(y)$ совпадут, не превосходит $1/m$.

**Замечание.** Определение универсального семейства говорит только о совпадении значений хэш-функции двух элементов. Про вероятность совпадения значений трёх и более элементов в определении ничего не говорится.

**Пример 1.** Множество всех функций из $U$ в $\{0, \dots, m-1\}$ — универсальное семейство хэш-функций. Действительно, таких функций всего $m^{|U|}$, и значения $h(x)$ и $h(y)$ совпадут у $m^{|U|-1}$ из них, что и даёт искомую вероятность. Однако практической ценности у такого семейства нет: чтобы просто записать одну хэш-функцию, потребуется слишком много битов.

**Теорема 1 (Картер и Вегман, 1979).** Пусть $U = \{0, \dots, N-1\}$, и пусть $p \ge N$ — большое простое число. Тогда множество всех функций из $U$ в $\{0, \dots, m-1\}$ вида
$$h_{a,b}(x) = ((ax + b) \bmod p) \bmod m,$$
где $a \in \{1, \dots, p-1\}$ и $b \in \{0, \dots, p-1\}$, — универсальное семейство хэш-функций.

**Доказательство.** Пусть значения хэш-функции совпали для двух разных аргументов: $h_{a,b}(x) = h_{a,b}(y)$ для $x \ne y$. Так как $x$ и $y$ различны и не превосходят $p$, линейные выражения дадут разные остатки по простому модулю: $ax + b \not\equiv ay + b \pmod p$. Поэтому совпадение получается лишь при взятии остатков по следующему модулю: $((ax + b) \bmod p) \equiv ((ay + b) \bmod p) \pmod m$.

Пусть $ax + b \equiv i \pmod p$ и $ay + b \equiv j \pmod p$, где $i \ne j$. Это система уравнений в поле $\mathbb{Z}_p$ с двумя неизвестными $a$ и $b$, и для всяких $i, j$ у неё будет единственное решение: $a = (i - j)(x - y)^{-1}$ и $b = i - ax$, где все действия выполняются в $\mathbb{Z}_p$. То есть парам $(i, j)$, где $i \ne j$, взаимно однозначно сопоставлены пары $(a, b)$, где $a \ne 0$ — всего $p(p-1)$ пар.

Утверждается, что среди всех пар $(i, j) \in \{0, \dots, p-1\}$, где $i \ne j$, таких пар, что $i \equiv j \pmod m$, не более чем $p \cdot \lfloor (p-1)/m \rfloor \le p \cdot \frac{p-1}{m} = \frac{p(p-1)}{m}$. Поэтому при случайном выборе $(a, b)$ вероятность попасть в такую пару не превосходит $1/m$. ■

**Семейство multiply-shift (2-универсальное) (из `Лекция 11.txt`)** 

Пусть ключи — $w$-битовые целые числа, т.е. $U = \{0, 1, \dots, 2^w-1\}$. Пусть $m = 2^\ell$. Зададим функцию $h_a(x) = ((a \cdot x) \gg (w-\ell)) \ \&\ (2^\ell - 1)$, где $a$ — нечётное $w$-битовое число. $H = \{h_a : a \text{ — нечётное } w\text{-битовое число}\}$.

**Доказательство.** Обозначим $m = 2^t$. Пусть $x, y \in U$, $x < y$. $h_a(x) = h_a(y) \iff ax \bmod 2^w$ и $ay \bmod 2^w$ имеют одинаковые старшие $t$ битов. Следовательно, $a(y-x) \bmod 2^w$ имеет все старшие $t$ битов $00\dots0$ или $11\dots1$.

Обозначим $y - x = z \cdot 2^b$, где $z$ нечётное и $b \ge 0$. Число $a$ нечётное.

- **Случай $b > w - t$.** Тогда при любом $a$ число $a(y-x) \bmod 2^w$ содержит в своих старших $t$ битах и нули, и единицы.
- **Случай $b \le w - t$.** $a(y-x) = ****rrrrrrr10000$ в битовом представлении, где $rrrrrrr$ обозначают $w-1$ битов, следующих после самой младшей единицы.

Докажем, что при случайном равновероятном выборе нечётного $a$ биты $rrrrrrr$ случайно и равновероятно принимают все возможные $2^{w-1}$ значений. $a = 2a' + 1$, где $a'$ — случайное $w-1$-битовое число. Если $a'$ и $a''$ различны, то $a'z \ne a''z \pmod{2^{w-1}}$, так как $z$ нечётно. Следовательно, отображение $a' \mapsto (a'z) \bmod 2^{w-1}$ — биекция. Поэтому $a'z$ имеет случайные и равновероятные младшие $w-1$ битов. Сумма случайных битов с фиксированными битами числа $z$ тоже даёт случайные биты.

**Хеш-таблица с разрешением коллизий цепочками (из `Лекция 11.txt`)**

Таблица $T[0..m-1]$, каждая ячейка содержит связный список ключей, которые попали в эту ячейку. Если $n > \alpha m$, где $\alpha$ — некоторое число, то таблица увеличивается и перестраивается.

**Теорема (из `Лекция 11.txt`).** Ожидаемое время работы операций равно $O(1)$ (без учёта времени перестройки таблицы), если $h$ выбирается случайно равновероятно из $c$-универсального семейства функций $H$, где $c = O(1)$.

**Доказательство.** Достаточно доказать, что ожидаемая длина списка $T[h(x)]$, в который попадает $x$, не более $O(1)$. Для каждого ключа $y$ из таблицы создаём индикаторную случайную величину $I_y = [h(y) = h(x)]$. Из свойства $c$-универсальности имеем $\Pr(I_y = 1) \le c/m$. Отсюда ожидаемая длина списка $\mathbb{E}[\sum_y I_y] \le c n / m \le c \alpha \le O(1)$. ■




### 15. Совершенное хеширование. Двухуровневое совершенное хеширование.

**Совершенное хеширование (Static) (из `okhotin_algorithms3_2025_l5.pdf`, стр. 3-4)**

Пусть множество элементов $S \subseteq U$ дано заранее, и нужно построить структуру данных для его хранения, обеспечивающую эффективный поиск — всегда за время $O(1)$, даже в худшем случае. Одновременно нужно использовать как можно меньше памяти. Пусть $N = |S|$.

В идеальном случае хотелось бы найти хэш-функцию $h: U \to \{0, \dots, N-1\}$, которая действовала бы на $S$ взаимно однозначно. Но эффективного построения такой функции в общем случае нет.

**Лемма 1 (из `okhotin_algorithms3_2025_l5.pdf`, стр. 4).** Пусть $S \subseteq U$, $N = |S|$. Пусть $H$ — универсальное семейство хэш-функций из $U$ в $\{0, \dots, m-1\}$, где $m = N^2$. Пусть $h \in H$ выбирается случайно. Тогда $h$ отображает разные элементы $S$ в разные значения с вероятностью не меньше $1/2$.

**Доказательство.** Всего неупорядоченных пар элементов $x, y \in S$, $x \ne y$, ровно $\binom{N}{2}$. Для всякой такой пары вероятность получить одинаковые значения хэш-функции не превосходит $1/m = 1/N^2$ по определению универсального семейства. Поэтому вероятность совпадения значений хэш-функции для хотя бы одной пары не превосходит
$$\binom{N}{2} \cdot \frac{1}{N^2} = \frac{N(N-1)}{2} \cdot \frac{1}{N^2} \le \frac{1}{2}.$$

**Двухуровневая схема Фредмана–Комлоша–Семереди (1984).**

Сперва выбирается хэш-функция $h$ из универсального семейства $H$ хэш-функций из $U$ в $\{0, \dots, N-1\}$, где $N = |S|$. Тогда некоторые значения хэш-функции могут использоваться для нескольких элементов $S$. Для всякого $i \in \{0, \dots, N-1\}$ пусть $S_i = \{x \in S \mid h(x) = i\}$ — все элементы со значением $i$, и пусть $n_i = |S_i|$ — их количество, так что $\sum_i n_i = N$. Для каждого такого значения $i$ по лемме 1 строится своя хэш-таблица с $n_i^2$ ячеек. Всего потребуется $\sum_i n_i^2$ ячеек.

**Лемма 2 (из `okhotin_algorithms3_2025_l5.pdf`, стр. 4).** При случайном выборе хэш-функции $h \in H$ математическое ожидание суммарного размера таблиц $\sum_i n_i^2$ не превосходит $2N - 1$.

**Доказательство.** Для всякого $m$ выполняется $m^2 = m + m(m-1) = m + 2\binom{m}{2}$. Тогда сумму $\sum_i n_i^2$ можно представить в виде $\sum_i (n_i + 2\binom{n_i}{2}) = N + 2\sum_i \binom{n_i}{2}$. Её математическое ожидание тогда равно
$$\mathbb{E}\sum_i n_i^2 = N + 2\mathbb{E}\sum_i \binom{n_i}{2}.$$
Но оставшаяся сумма $\sum_i \binom{n_i}{2}$ — это количество неупорядоченных пар различных элементов $x, y \in S$ с совпадающими значениями хэш-функции: $h(x) = h(y)$. Поскольку хэш-функция $h$ выбрана случайно из универсального семейства, каждое такое совпадение происходит с вероятностью не более $1/N$, и математическое ожидание их суммы не превосходит $\binom{N}{2} \cdot \frac{1}{N}$:
$$\mathbb{E}\sum_i n_i^2 = N + 2 \cdot \binom{N}{2} \cdot \frac{1}{N} = N + N(N-1)\frac{1}{N} = 2N - 1.$$

Отсюда, по неравенству Маркова, суммарный размер таблиц $\sum_i n_i^2$ не превосходит $4N$ с вероятностью не менее $1/2$. Если получится больше, можно выбрать новые случайные хэш-функции.

В полученной структуре данных для поиска элемента нужно вычислить одно или два значения хэш-функций — время $O(1)$.


### 16. Задача проверки простоты числа. Тест Ферма. Числа Кармайкла. Алгоритм Миллера–Рабина.

**Тест Ферма (вероятностный) (из `okhotin_algorithms3_2025_l6.pdf`, стр. 2)**

Основан на малой теореме Ферма: для всякого простого числа $p$ равенство $a^p \equiv a \pmod p$ верно для всех $a \ge 1$, или в эквивалентной форме $a^{p-1} \equiv 1 \pmod p$ для $a$, не кратных $p$.

**Algorithm 15. Вероятностная проверка простоты числа по условию Ферма**
```
Algorithm FermatTest(n)
    Input: натуральное число n
    Output: «составное» или «возможно, простое»

1:  Пусть a ∈ {2, . . . , n − 2} — случайное число
2:  if a^{n−1} ≠ 1 (mod n) then
3:      return «составное»
4:  else
5:      return «возможно, простое»
```

Если $n$ простое, то, согласно малой теореме Ферма, алгоритм всегда даёт ответ «возможно, простое». Если же $n$ составное, то алгоритм может дать как ответ «составное», так и «возможно, простое».

**Числа Кармайкла.** Существуют составные числа $n$, для которых $a^{n-1} \equiv 1 \pmod n$ для всех $a$, взаимно простых с $n$ (последовательность A002997 в OEIS). Наименьшее число Кармайкла — $561 = 3 \cdot 11 \cdot 17$. Если алгоритму 15 подаётся на вход число Кармайкла, он может определить, что оно составное, только если ему посчастливится выбрать один из его делителей. Но вероятность этого мала, и потому алгоритм будет почти всегда давать неправильный ответ.

**Algorithm 16. Алгоритм Миллера–Рабина (из `okhotin_algorithms3_2025_l6.pdf`, стр. 4)**
```
Algorithm MillerRabin(n, k)
    Input: натуральное число n, параметр точности k
    Output: «составное» или «возможно, простое»

1:  for j = 1 to k do
2:      пусть a ∈ {2, . . ., n − 1} — случайное число
3:      пусть n − 1 = 2^t·u, где t ≥ 0 и u нечётно
4:      x₀ = a^u (mod n)
5:      for i = 1 to t do
6:          x_i = x_{i-1}^2 (mod n)
7:          if x_i ≡ 1 и x_{i-1} ≠ 1 и x_{i-1} ≠ n − 1 (mod n) then
8:              return «составное»
9:      if x_t ≠ 1 then
10:         return «составное»
11: return «возможно, простое»
```

**Теорема 2 (из `okhotin_algorithms3_2025_l6.pdf`, стр. 4).** Если алгоритм Миллера–Рабина даёт ответ «составное», то число действительно составное. Если же число составное, то он даёт ответ «возможно, простое» с вероятностью менее $1/4^k$.

**Доказательство.** Ответ «составное» выдаётся в одном из двух случаев:
1. $a^{2^{i-1}u} \not\equiv 1, n-1 \pmod n$, но $a^{2^i u} \equiv 1 \pmod n$ — найден контрпример к условию леммы о квадратных корнях из единицы, свидетельствующий о том, что $n$ не может быть ни простым числом, ни даже степенью простого.
2. $a^{n-1} \not\equiv 1 \pmod n$ — контрпример к условию малой теоремы Ферма.

Осталось оценить вероятность ошибки — то есть вероятность случая, что число $n$ составное, но для $a$ даётся ответ «возможно, простое». В этом случае $a$ будет взаимно простым с $n$ (иначе $a^{n-1}$ было бы кратно общему делителю).

**Случай 1:** Существует $b$, взаимно простое с $n$, для которого $b^{n-1} \not\equiv 1 \pmod n$ (т.е. $n$ — не число Кармайкла). Множество всех $a \in \mathbb{Z}_n^*$, для которых $a^{n-1} \equiv 1 \pmod n$, образует подгруппу группы $\mathbb{Z}_n^*$. По теореме Лагранжа число элементов в подгруппе не больше $|\mathbb{Z}_n^*|/2$. Поэтому такое $a$ будет выбрано с вероятностью $\le 1/2$.

**Случай 2:** Для всех $b$, взаимно простых с $n$, верно $b^{n-1} \equiv 1 \pmod n$ (т.е. $n$ — число Кармайкла). Пусть $n = n_1 n_2$, где $n_1, n_2 \ge 3$ взаимно просты. Рассматривается множество $X = \{x \in \mathbb{Z}_n^* \mid x^{2^j u} \equiv \pm 1 \pmod n\}$ для наибольшего $j$, для которого существует $c$ с $c^{2^j u} \equiv -1 \pmod n$. $X$ — подгруппа, и можно показать, что она не совпадает с $\mathbb{Z}_n^*$ (например, построив элемент $y$ с $y \equiv c \pmod{n_1}$, $y \equiv 1 \pmod{n_2}$). Следовательно, $|X| \le |\mathbb{Z}_n^*|/2$, и вероятность ошибки $\le 1/2$.

Повторяя тест $k$ раз, получаем вероятность ошибки $\le (1/4)^k$, так как в каждом раунде ошибка может произойти с вероятностью не более $1/4$ (учитывая оба случая). ■

**Время работы:** пусть $N$ — количество разрядов в данном числе $n$. Одно умножение двух $N$-разрядных чисел по модулю $n$ потребует порядка $N(\log N)^{O(1)}$ операций. Всего таких умножений потребуется порядка $O(N)$, и потому время работы алгоритма Миллера–Рабина составляет $O(k \cdot (\log n)^3)$.


### 17. Онлайн-алгоритмы. Задача кэширования. Помечающий алгоритм.

**Онлайн-алгоритмы (из `okhotin_algorithms3_2025_l6.pdf`, стр. 6)**

Алгоритм, обрабатывающий вход по мере его поступления (online algorithm), получает входную строку символ за символом и должен выдавать какие-то ответы после каждого куска данных, не дожидаясь следующего куска. По сравнению с таким алгоритмом, обычный алгоритм, получающий сразу всю строку (offline algorithm), знает будущее.

Качество онлайн-алгоритма оценивается **конкурентным анализом**: коэффициент конкурентноспособности $\rho$ означает, что для любой входной последовательности стоимость алгоритма $\le \rho \cdot OPT$, где $OPT$ — стоимость оптимального офлайн алгоритма, знающего всю последовательность.

**Задача кэширования страниц памяти (paging) (из `okhotin_algorithms3_2025_l6.pdf`, стр. 6-7)**

Всего страниц $n$, в кэш влезают $k$ страниц, где $k < n$. На вход поступает последовательность страниц. Если поступающая очередная страница есть в кэше (попадание), стоимость этого шага — 0. Если нет (промах), то её надо переместить в кэш, убрав оттуда какую-то из находящихся в нём страниц; стоимость — 1. На каждом шаге вопрос в том, какую страницу убирать.

**Помечающий алгоритм (marking algorithm) (из `okhotin_algorithms3_2025_l6.pdf`, стр. 7)**

Каждая страница в кэше может быть помечена — пометка означает защиту от вытеснения. Изначально в кэше лежат произвольные страницы, все не помечены.

Работа алгоритма разделена на периоды: в начале каждого периода все страницы не помечены. Если новая страница есть в кэше, она помечается. Если её нет, но есть непомеченные, то удаляется случайно выбранная непомеченная, а новая помечается. Если все помечены, то сперва все пометки снимаются, и тем самым начинается новый период. Таким образом, в начале каждого периода в памяти будут те и только те страницы, к которым в предыдущем периоде были обращения.

**Algorithm 17. Помечающий алгоритм кэширования**
```
Algorithm MarkingPaging
    Input: последовательность обращений к страницам, размер кэша k
    Output: последовательность действий (выгрузки/загрузки)

1:  Инициализация: cache = ∅, marked = ∅
2:  for each page p in sequence do
3:      if p in cache then
4:          marked = marked ∪ {p}          // попадание: помечаем
5:      else
6:          // промах
7:          cache = cache ∪ {p}
8:          marked = marked ∪ {p}
9:          if |cache| > k then
10:             if ∃ непомеченная страница in cache then
11:                 выбрать случайную непомеченную страницу u
12:                 cache = cache \ {u}
13:             else
14:                 // все страницы помечены — начинаем новую фазу
15:                 marked = ∅
16:                 выбрать случайную страницу u ∈ cache \ {p}
17:                 cache = cache \ {u}
18:             // загрузка p уже выполнена на шагах 7-8
```

**Теорема 4 (Фиат и др., 1991).** Для всякого детерминированного алгоритма кэширования, знающего будущее, и для всякой входной последовательности математическое ожидание числа промахов помечающего алгоритма не более чем в $2H_k$ раз превышает число промахов детерминированного алгоритма на этом входе, где $H_k = \sum_{i=1}^k 1/i$ — $k$-е гармоническое число.

**Доказательство (схема).** Пусть $i$ — номер периода. Обозначим через $m_i$ количество новых страниц, запрошенных в $i$-м периоде. Математическое ожидание числа промахов в $i$-м периоде не превосходит $m_i H_k$. Для детерминированного алгоритма, знающего будущее, число промахов $n_i$ в $i$-м периоде удовлетворяет $n_{i-1} + n_i \ge m_i$ для $i \ge 2$, а $n_1 = k$. Суммируя, получаем $2 \sum_i n_i \ge \sum_i m_i$. Следовательно, отношение ожидаемого числа промахов помечающего алгоритма к числу промахов оптимального офлайн алгоритма не превосходит
$$\frac{\sum m_i H_k}{\frac12 \sum m_i} = 2H_k.$$

**Пояснение**

Промахи в периоде i:

1. обращения к новам страницам (их нет в кеше).
2. обращения к старым вытесненным страницам (их заметили новые страницы из кеша)

1. вероятность промаха на новах траницах - 1 $\mathrm{E}(\text{промахи на новых}) = 1$.
2. вероятность промаха на старых страницах оценим сверху

Вероятность промаха на старой страницах, не больше чем вероятность промаха на этой странице после добавления всех НОВЫХ (банально, чем больше вытеснили, тем больше вероятность промаха).

пусть сначала добавили все $m_i$ новых страниц и вытеснили $m_i$ старых.

вероятность промаха на i старой - $P_i = \frac{m_i}{k-i}$ без учета $i = k - 1$

Случай $i = k - 1$:

все страницы старые, значит последняя не вытеснена, вероятность промаха = 0.

Итого,

$$
\mathrm{E}(\# \text{промах}) \le m_i \left ( \frac{1}{k} + \frac{1}{k-1} + \dots + \frac{1}{2} \right )  + E(\# \text{промах на новых}) = m_i H_k
$$

Теперь про неравенство.

За 2 периода суммарно должно быть загружено $k + m_i$.

За два периода промахов должно быть не $\le m_i$


### 18. Задача нахождения минимального остовного дерева. Алгоритмы Ярника–Прима–Дейкстры и Краскала. Системы непересекающихся множеств (Union-Find).

**Задача**

Дан неориентированный связный граф $G = (V, E)$, рёбрам которого сопоставлены числа — веса $w: E \to \mathbb{R}$. Нужно найти одно из остовных деревьев с наименьшим весом.

**Общий принцип алгоритмов:** одно за другим строятся рёбра, принадлежащие некоторому минимальному остовному дереву; каждое добавленное ребро — правильное. Такие алгоритмы называются «жадными» (greedy).

**Какое ребро можно добавить? (Лемма 4 из `okhotin_algorithms3_2025_l3.pdf`, стр. 7)**

Сечение графа $G = (V, E)$ — это всякое разбиение множества вершин на $C$ и $V \setminus C$. Ребро $(u, v) \in E$ пересекает сечение, если один его конец лежит в $C$, а другой — в $V \setminus C$.

**Лемма.** Пусть $G = (V, E)$ — граф, $T \subseteq E$ — одно из минимальных остовных деревьев, $F \subseteq T$ — его подмножество. Пусть $(C, V \setminus C)$ — некоторое сечение графа, такое что никакое ребро из $F$ его не пересекает. Тогда ребро с наименьшим весом, пересекающее это сечение, принадлежит некоторому минимальному остовному дереву $T'$, также содержащему $F$.

**Доказательство.** $\triangleright$

Пусть дано разбиение $C$.

$$
\forall uv \in \text{сечение}  \notin E(F) \Rightarrow F = F|_C \sqcup F | _{V \setminus C}
$$

$$
Let \ uv - min \{E_{\text{сечение}} \}
$$, добавим его в наш T

у нас образуется цикл $e_1 xy e_2 uv :$

$$
e_1, \in E(C), \ e_2 \in E(V \setminus C) \\ xy, \ uv \in E(\text{сечение})
$$

можем просто удалить $xy$. 

Минимальность очевидна.

**Algorithm 18. Алгоритм Прима (Ярника–Прима–Дейкстры) (из `okhotin_algorithms3_2025_l3.pdf`, стр. 8-9)**
```
Algorithm Prim(G, s)
    Input: граф G = (V, E) с весами, стартовая вершина s
    Output: минимальное остовное дерево T

1:  d_s = 0
2:  for all v ≠ s do d_v = ∞
3:  T = ∅
4:  Q = V          // очередь с приоритетами по d[v]
5:  while Q ≠ ∅ do
6:      u = arg min_{v∈Q} d_v   // извлекаем вершину с минимальным d
7:      Q = Q \ {u}
8:      if u ≠ s then
9:          T = T ∪ {(u, π_u)}
10:     for all v: (u, v) ∈ E do
11:         if v ∈ Q and d_v > w_{u,v} then
12:             d_v = w_{u,v}
13:             π_v = u
14: return T
```

**Время работы:** $O(|E| \log |V|)$ (с использованием двоичной кучи).

**Algorithm 19. Алгоритм Краскала (из `okhotin_algorithms3_2025_l4.pdf`, стр. 2)**
```
Algorithm Kruskal(G)
    Input: граф G = (V, E) с весами
    Output: минимальное остовное дерево T

1:  T = ∅
2:  for each vertex v ∈ V do
3:      make_set(v)
4:  отсортировать рёбра E в порядке увеличения их веса
5:  for each (u, v) ∈ E в порядке увеличения веса do
6:      if find_set(u) ≠ find_set(v) then
7:          T = T ∪ {(u, v)}
8:          union(u, v)
9:  return T
```

**Время работы:** $O(|E| \log |E|) = O(|E| \log |V|)$ (доминирует сортировка).

**Система непересекающихся множеств (Union-Find) (из `okhotin_algorithms3_2025_l4.pdf`, стр. 2-3)**

Абстрактная структура данных для представления разбиения множества на непересекающиеся подмножества. У каждого множества есть выделенный элемент — «представитель». Операции:
- `make_set(x)`: создать одноэлементное множество $\{x\}$.
- `find_set(x)`: найти представителя множества, содержащего элемент $x$.
- `union(x, y)`: объединить два множества, содержащие $x$ и $y$.

**Эффективная реализация (лес непересекающихся множеств).** Это лес из деревьев с путями, ведущими к корню. Каждое дерево соответствует одному из хранимых подмножеств, вершины дерева — его элементам. Элемент в корне дерева называется представителем множества.

**Эвристики:**
1. **Объединение по рангу.** В каждой вершине хранится её условная сложность — ранг. Процедура `make_set(x)` устанавливает ранг в ноль. При объединении двух деревьев корень дерева меньшего ранга становится указывать на корень дерева большего ранга. Если оба корня имеют одинаковый ранг, то их объединение получает на единицу больший ранг.
2. **Сжатие пути.** При выполнении каждой операции поиска все встреченные на пути вершины перенаправляются в корень для ускорения последующих операций поиска.

**Время работы.** Пусть всего элементов — $n$, и к ним применяется $m$ операций. С применением объединения по рангу операции выполняются за время $O(m \log n)$. С обеими эвристиками — почти линейное время: $O(m \cdot \alpha(n))$, где $\alpha(n)$ — обратная функция Аккермана (растёт крайне медленно, $\le 4$ для практических $n$).


### 19. Задача булевой выполнимости (SAT). Задачи kSAT и MAX-kSAT. Вероятностный и детерминированный приближённые алгоритмы для MAX-3SAT.

**Задача SAT**

**Задача:** Дана булева формула в КНФ (конъюнкция дизъюнкций). Существует ли набор переменных, делающий формулу истинной?

**$k$SAT:** Ограничение, что каждый дизъюнкт (clause) содержит не более $k$ литералов. $k$SAT NP-полна для $k \ge 3$.

**MAX-$k$SAT:** Оптимизационная задача. Найти набор переменных, максимизирующий количество **выполненных** (истинных) дизъюнктов.

**Вероятностный алгоритм для MAX-3SAT (из `okhotin_algorithms3_2025_l3.pdf`, стр. 3-4)**

**Лемма 1.** Во всякой 3-КНФ с $m$ дизъюнкциями математическое ожидание количества истинных дизъюнкций при случайном выборе значений переменных равно $\frac{7}{8}m$.

**Доказательство.** В отдельности каждая дизъюнкция будет истинной с вероятностью $1 - (1/2)^3 = 7/8$. По линейности математического ожидания получаем утверждение.

**Лемма 2.** Во всякой 3-КНФ с $m$ дизъюнкциями при случайном выборе значений переменных хотя бы $\frac{7}{8}m$ дизъюнкций будут истинными с вероятностью не менее $\frac{1}{8m}$.

**Доказательство.** Пусть $p_i$ — вероятность, что истинными будут ровно $i$ дизъюнкций, $p = \sum_{i \ge 7m/8} p_i$. Тогда
$$\frac{7}{8}m = \sum_{i < 7m/8} i p_i + \sum_{i \ge 7m/8} i p_i \le \left(\frac{7}{8}m - \frac{1}{8}\right)(1-p) + m p.$$
Решая неравенство, получаем $p \ge \frac{1}{8m}$.

**Algorithm 20. Вероятностный алгоритм для MAX-3SAT**
```
Algorithm Max3SatRandom(formula)
    Input: 3-КНФ формула с m дизъюнктами
    Output: набор значений переменных

1:  for i = 1 to 8m do
2:      случайно выбрать значения всех переменных (0 или 1 с вероятностью 1/2)
3:      вычислить количество выполненных дизъюнктов
4:      запомнить лучший найденный набор
5:  return лучший набор
```

Алгоритм является $\frac{8}{7}$-приближением, так как $\frac{8}{7} \cdot \frac{7}{8}m = m$.

**Детерминированный алгоритм (дерандомизация) (из `okhotin_algorithms3_2025_l3.pdf`, стр. 5-6)**

Метод условных математических ожиданий. Переменные назначаются последовательно. На каждом шаге вычисляется матожидание числа выполненных дизъюнктов при условии уже сделанных назначений. Выбирается то значение (0 или 1), которое даёт большее матожидание. Так как изначально матожидание было $7m/8$, а в конце оно становится равным фактическому числу выполненных дизъюнктов, алгоритм гарантированно находит набор с $\ge 7m/8$ истинными дизъюнктами.

**Вычисление матожидания.** Для невыполненного и неопределённого дизъюнкта из $t$ неопределённых переменных вероятность его выполнения равна $1 - (1/2)^t$. Этому можно присвоить вес.

**Теорема 2 (Джонсон, 1974).** Существует детерминированный $\frac{8}{7}$-приближённый алгоритм для задачи MAX-3SAT, работающий за полиномиальное время.

**Результат о неаппроксимируемости (Хостад, 2001).** Существование $(\frac{8}{7} - \varepsilon)$-приближённого алгоритма для MAX-3SAT эквивалентно $P = NP$.
