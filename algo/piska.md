# Ответы на экзаменационные вопросы по курсу «Математические основы алгоритмов»
## Весенний семестр 2026 г. (группы ИИ и СП)

### 1. Дискретное преобразование Фурье (DFT), обратное DFT. Свертки и умножение многочленов при помощи алгоритма FFT. Обобщения на другие кольца, NTT. Вычислительная схема FFT (преобразования бабочки).

**Определения**
*   **Свертка (конволюция)** последовательностей $a_0, \dots, a_{n-1}$ и $b_0, \dots, b_{n-1}$ — это последовательность $c_0, \dots, c_{2n-2}$, где $c_k = \sum_{t=0}^{n-1} a_t b_{k-t}$ (считая $a_i = b_i = 0$ при $i \notin [0, n-1]$).
*   **Умножение многочленов** $A(x) = \sum a_i x^i$ и $B(x) = \sum b_i x^i$ дает многочлен $C(x) = A(x)B(x)$, коэффициенты которого являются сверткой коэффициентов $A$ и $B$.
*   **Дискретное преобразование Фурье (DFT)**: для вектора $\mathbf{a} = (a_0, \dots, a_{N-1})$ и примитивного корня $N$-й степени из единицы $\omega$ (например, $\omega = e^{2\pi i / N}$) определяется как $\mathbf{y} = F_\omega \mathbf{a}$, где $y_k = \sum_{j=0}^{N-1} a_j (\omega^k)^j$.

**Основная идея FFT**
Наивное умножение многочленов требует $O(n^2)$ операций. FFT позволяет сделать это за $O(n \log n)$, используя следующий подход (интерполяция):
1.  Выбрать $N \ge 2n-1$ (обычно $N = 2^t$) и дополнить коэффициенты $A$ и $B$ нулями до длины $N$.
2.  Вычислить значения $A(\omega^k)$ и $B(\omega^k)$ для $k = 0, \dots, N-1$ с помощью **прямого FFT**.
3.  Покомпонентно перемножить значения: $y_k = A(\omega^k) B(\omega^k)$.
4.  По значениям $y_k$ восстановить коэффициенты многочлена $C$ с помощью **обратного FFT**.

**Реализация FFT (рекурсивная "бабочка")**
Идея рекурсивного разбиения ("разделяй и властвуй"): разбить $A(x)$ на четные и нечетные коэффициенты:
*   $A_0(x) = a_0 + a_2 x + \dots + a_{N-2} x^{N/2-1}$
*   $A_1(x) = a_1 + a_3 x + \dots + a_{N-1} x^{N/2-1}$

Тогда $A(x) = A_0(x^2) + x A_1(x^2)$. Подставляя $x = \omega^k$, получаем:
$$A(\omega^k) = A_0(\omega^{2k}) + \omega^k A_1(\omega^{2k})$$
Так как $\omega^{2k}$ пробегают лишь $N/2$ различных значений, задача сводится к двум подзадачам размера $N/2$. Склейка результатов ("бабочка"):
*   $y_k = y_k^{(0)} + \omega^k y_k^{(1)}$
*   $y_{k+N/2} = y_k^{(0)} - \omega^k y_k^{(1)}$ (так как $\omega^{k+N/2} = -\omega^k$)

**Обратное FFT**
Матрица DFT $F_\omega$ — это матрица Вандермонда. Можно показать, что $F_\omega^{-1} = \frac{1}{N} F_{\omega^{-1}}$. То есть обратное преобразование — это прямое FFT с корнем $\omega^{-1}$, после чего каждый коэффициент делится на $N$.

**Вычислительная схема FFT**
Схема FFT имеет рекурсивную структуру: два блока $FFT_{N/2}$, к выходам которых добавляется слой из $N/2$ "бабочек". Бабочка — элементарный шаг, который умножает один вход на $\omega^k$ и выполняет попарное сложение и вычитание.
*   **Глубина схемы**: $O(\log N)$.
*   **Размер схемы**: $O(N \log N)$.

**Обобщения (NTT)**
FFT работает в любом кольце/поле, где:
1.  Существует примитивный корень $\omega$ порядка $N$.
2.  $N$ обратимо в кольце (т.е. можно делить на $N$ для обратного преобразования).
*   **Теоретико-числовое преобразование (NTT)**: Работает в поле $\mathbb{Z}_p$ (остатки по модулю простого числа $p$), где $p = c \cdot 2^k + 1$. Тогда существует первообразный корень $g$ по модулю $p$, и можно взять $\omega \equiv g^c \pmod{p}$, который будет иметь порядок $N = 2^k$. NTT позволяет избежать ошибок округления, присущих работе с комплексными числами.

### 2. Вычисления схемами как модель параллельных вычислений. Компараторы, сортирующие сети. Принцип нулей-единиц. Битонная сортировка и сортировка чётно-нечётным слиянием. (На основе конспекта Косолобова)

**Вычислительная схема**
Это ациклический ориентированный граф, где:
*   **Входные вершины** имеют степень входа 0.
*   **Выходные вершины** имеют степень исхода 0.
*   **Внутренние вершины** помечены элементарной операцией (арифметической, булевой и т.д.).
*   **Размер схемы** — число внутренних вершин.
*   **Глубина схемы** — длина максимального пути от входа к выходу (параллельное время вычисления).

**Сортирующие сети**
Это схема, которая сортирует последовательность чисел, используя в качестве внутренних элементов только **компараторы**. Компаратор с входами $u, v$ выдает $u' = \min(u, v)$, $v' = \max(u, v)$.

**Принцип нулей и единиц (Теорема 2.1)**
Сортирующая сеть корректна для всех вещественных входов **тогда и только тогда**, когда она корректна для всех входов из нулей и единиц.
*   *Идея доказательства*: Если сеть ошибается на вещественных числах ( $y_i > y_j$ ), то выбирается порог $t$ между $y_j$ и $y_i$. Применение этого порога ко всем элементам на входе и выходе коммутирует с работой компараторов. Полученная булева последовательность будет неотсортированной, что противоречит условию.

**Битонная сортировка**
*   **Битонная последовательность**: Циклический сдвиг последовательности, которая сначала не убывает, а затем не возрастает (например, $0^i 1^j 0^k$).
*   **Алгоритм** (для $n = 2^t$):
    1.  Рекурсивно отсортировать левую и правую половины по возрастанию.
    2.  Конкатенация (отсортированная левая половина, перевернутая отсортированная правая половина) является битонной.
    3.  Применить к полученной последовательности **битонное слияние** (`BI_MERGE`).
*   **Битонное слияние** (`BI_MERGE`):
    *   Сравнивает элементы на расстоянии $m/2$: $(z_k, z_{k+m/2})$, формируя последовательности минимумов и максимумов.
    *   Рекурсивно сливает обе полученные последовательности.
    *   **Сложность**: Глубина $O(\log m)$, размер $O(m \log m)$.
*   **Сложность всей сортировки**: Глубина $O(\log^2 n)$, размер $O(n \log^2 n)$.

**Сортировка четно-нечетным слиянием (Бачера)**
*   **Идея**: Рекурсивно отсортировать половины, а затем слить их алгоритмом `ODD_EVEN_MERGE`.
*   **Слияние** (`ODD_EVEN_MERGE`):
    1.  Рекурсивно слить элементы на нечетных позициях двух отсортированных массивов и отдельно — на четных.
    2.  Скомбинировать результаты, используя финальный слой компараторов для исправления возможных инверсий.
*   **Сложность**: Такая же, как у битонной сортировки: глубина $O(\log^2 n)$ и размер $O(n \log^2 n)$.

*(Примечание: в конспекте упоминается сеть AKS с глубиной $O(\log n)$ и размером $O(n \log n)$, но она значительно сложнее.)*

### 3. Префиксные суммы (scan), схема Ладнера–Фишера. Модель CRCW/EREW PRAM. Сортировка подсчётом в модели PRAM.

**Префиксные суммы (scan)**
Задача: по массиву $x_0, \dots, x_{n-1}$ вычислить массив $y_i = \sum_{j=0}^i x_j$ для ассоциативной операции (сумма, min, max и т.д.).

**Схема Ладнера–Фишера**
Идея "подъема и спуска" (upsweep-downsweep):
1.  **Свертка (upsweep)**: Попарно складываем соседние элементы: $s_i = x_{2i} + x_{2i+1}$.
2.  **Рекурсия**: Рекурсивно вычисляем префиксные суммы $p_i$ для массива $s$.
3.  **Развертка (downsweep)**: Восстанавливаем ответ: $y_{2i+1} = p_i$, $y_{2i} = p_{i-1} + x_{2i}$ (для $i \ge 1$).
*   **Глубина**: $T(n) = 2 + T(n/2) = O(\log n)$.
*   **Размер**: $S(n) = (n-1) + S(n/2) = O(n)$.

**Модель PRAM**
Модель параллельных вычислений с общей памятью, синхронными процессорами и двумя режимами доступа:
*   **EREW (Exclusive Read Exclusive Write)**: Одновременное чтение и запись в одну ячейку запрещены.
*   **CREW (Concurrent Read Exclusive Write)**: Чтение разрешено, запись — нет.
*   **CRCW (Concurrent Read Concurrent Write)**: Чтение и запись разрешены (для записи нужно правило разрешения конфликта, например, arbitrary).
*   **Класс NC**: Задачи, решаемые на PRAM за $(\log n)^{O(1)}$ времени с $n^{O(1)}$ процессорами.

**Сортировка подсчетом в PRAM** (Самописное, на основе конспекта. Правила: массив $a[0..n-1]$, $a[k] \in \{0, \dots, U\}$, $U = O(1)$.)
1.  **Локальный подсчет**: $p$ процессоров обрабатывают свои блоки длины $n/p$, строя локальные таблицы частот $c_i$. Время: $O(n/p)$.
2.  **Сборка и scan**: Формируется массив $C$ длины $(U+1)p$, где $C[xp + i] = c_i(x)$. К нему применяется scan (префиксная сумма), чтобы для каждого процессора и значения получить глобальную начальную позицию в выходном массиве. Время: $O(\log p)$ (scan на длине $O(p)$, так как $U=O(1)$).
3.  **Расстановка**: Каждый процессор снова проходит по своему блоку и расставляет элементы в выходном массиве согласно вычисленным позициям. Время: $O(n/p)$.
*   **Итоговое время**: $O(n/p + \log p)$. Если выбрать $p = \Theta(n / \log n)$, то время будет $O(\log n)$.

### 4. Трудоемкость локальных вычислений, коммуникации и синхронизации при вычислениях в модели барьерно-синхронного параллелизма (BSP). Префиксные суммы в BSP, оптимальность. (На основе конспекта Косолобова)

**Модель BSP (Bulk Synchronous Parallel)**
Задается параметрами $p$ (число процессоров), $g$ (время передачи одного слова), $L$ (время синхронизации).
*   **Вычислительная модель**: $p$ компьютеров (RAM) с локальной памятью, соединенных сетью.
*   **Супершаг**: Алгоритм выполняется по супершагам, каждый из которых состоит из трех фаз:
    1.  **Локальные вычисления** на каждом компьютере.
    2.  **Обмен сообщениями** между компьютерами.
    3.  **Синхронизация** (барьер), на которой все компьютеры ожидают друг друга и доставляются сообщения.
*   **Время супершага $s$**: $\max_k t_k^{(s)} + \max_k (g \cdot c_k^{(s)}) + L$, где $t_k^{(s)}$ — время вычислений, $c_k^{(s)}$ — количество слов, отправленных/полученных $k$-м компьютером.
*   **Общая стоимость**: $\sum_s \max_k t_k^{(s)} + g \sum_s \max_k c_k^{(s)} + \hat{S} L$.

**Префиксные суммы в BSP**
Для $n$ чисел и $p$ процессоров, где $n \ge p$ (и $p$ делит $n$):
1.  **Супершаг 1**: Чтение данных из внешней памяти. Коммуникация: $O(n/p)$.
2.  **Супершаг 2**: Вычисление суммы своего блока ($O(n/p)$) и отправка ее первому процессору. Коммуникация: $O(p)$.
3.  **Супершаг 3**: Первый процессор вычисляет префиксные суммы по суммам блоков ($O(p)$) и рассылает смещения. Коммуникация: $O(p)$.
4.  **Супершаг 4**: Каждый процессор вычисляет локальные префиксные суммы ($O(n/p)$), прибавляет смещение и записывает результат. Коммуникация: $O(n/p)$.
*   **Общая стоимость**: Вычисления — $O(n/p)$, Коммуникация — $O(n/p)$, Синхронизация — $O(1)$.
*   **Оптимальность**: Алгоритм асимптотически оптимален, так как каждое входное слово нужно хотя бы один раз прочитать ($\Omega(n/p)$ коммуникаций и вычислений).

### 5. Параллельное вычисление обобщенных линейных рекуррентных соотношений и параллельное сложение двоичных чисел при помощи задачи префиксного накопления. (На основе конспектов Косолобова и Охотина)

**Вычисление линейной рекурренты**
Дана рекуррента: $c_0 = 0$, $c_{k+1} = a_k + b_k c_k$. Ее можно представить в матричной форме:
$$C_{k+1} = C_k X_k, \quad \text{где } C_k = \begin{pmatrix} 1 & c_k \end{pmatrix}, \quad X_k = \begin{pmatrix} 1 & a_k \\ 0 & b_k \end{pmatrix}$$
Тогда $C_n = C_0 X_0 X_1 \dots X_{n-1}$. Префиксные произведения матриц $X_0, \dots, X_{n-1}$ можно вычислить с помощью операции **scan** (префиксного накопления), так как умножение матриц ассоциативно.
*   **Сложность**: $O(\log n)$ времени в параллельной модели.
*   **Обобщение**: Работает для любого полукольца $(S, \oplus, \odot)$.

**Параллельное сложение двоичных чисел (Сумматор Ладнера–Фишера)**
Даны биты $x_k$, $y_k$ (два $n$-битных числа). Нужно вычислить сумму $z = x+y$.
*   **Биты переноса**: $c_0 = 0$, $c_{k+1} = (x_k \land y_k) \lor (x_k \land c_k) \lor (y_k \land c_k)$.
*   **Упрощение**: Обозначим $a_k = x_k \land y_k$, $b_k = x_k \lor y_k$. Тогда $c_{k+1} = a_k \lor (b_k \land c_k)$.
*   **Сведение к scan**: Эта рекуррента имеет тот же матричный вид, что и обобщенная линейная рекуррента:
    $$C_{k+1} = C_k X_k, \quad \text{где } C_k = \begin{pmatrix} 1 & c_k \end{pmatrix}, \quad X_k = \begin{pmatrix} 1 & a_k \\ 0 & b_k \end{pmatrix}$$
    Операции выполняются в булевом полукольце $(\{0,1\}, \lor, \land)$.
*   **Результат**: Вычисление всех битов переноса (и, следовательно, суммы $z_k = x_k \oplus y_k \oplus c_k$) сводится к scan матриц $X_k$ и может быть выполнено за $O(\log n)$ времени в параллельной модели.

### 6. Алгоритмы во внешней памяти. Сортировка слиянием. Сортировка связного списка (list ranking). (На основе конспекта Косолобова)

**Модель внешней памяти**
Задается параметрами $B$ (размер блока) и $M$ (размер внутренней памяти, $M \gg B$).
*   Чтение/запись блока из $B$ подряд идущих слов из внешней памяти стоит 1 операцию I/O.
*   Вычисления над данными во внутренней памяти — бесплатно.
*   **Время работы** = количество операций I/O.

**Сортировка слиянием во внешней памяти**
*(Алгоритм, похожий на $M/B$-арную сортировку слиянием)*
1.  Разбить массив на $M/B$ подмассивов (они помещаются в память).
2.  Рекурсивно отсортировать каждый подмассив (база — подмассив, целиком помещающийся в память, сортируется во внутренней памяти).
3.  Слить отсортированные подмассивы: так как все $M/B$ подмассивов могут предоставить по одному блоку во внутреннюю память, слияние $M/B$ потоков может идти без дополнительных I/O, за исключением подгрузки следующих блоков.
*   **Сложность**: $T(N) = O\left(\frac{N}{B} \log_{M/B} \frac{N}{B}\right) = sort(N)$.

**Сортировка связного списка (List Ranking)**
Задача: для каждой вершины списка вычислить ее номер (или сумму весов на префиксе). Идея рекурсивного случайного сжатия:
1.  Каждой вершине $x$ присваивается случайный бит $x.r \in \{0,1\}$.
2.  Во множество $S$ удаляемых вершин попадают те, у которых $x.r = 1$ и $pred(x).r = 0$. Такие вершины не являются соседними, $|S| = \Theta(N)$ в среднем.
3.  Для каждого $x \in S$: $pred(x).w \mathrel{+}= x.w$, $pred(x).next = x.next$ (удаление).
4.  Рекурсивно решить задачу для сжатого списка $A'$ длины $N - |S|$.
5.  При возврате из рекурсии восстановить удаленные вершины, обновив их веса: $x.w \mathrel{+}= pred(x).w$.
*   **Реализация во внешней памяти**: Все операции (выбор $S$, удаление, упаковка, восстановление) реализуются через константное число сортировок ($sort(N)$) и линейных проходов (scan) над массивами, а не через случайные переходы по указателям.
*   **Сложность**: Ожидаемое время работы $O(sort(N))$.

### 7. Задача линейного программирования. Допустимые и оптимальные решения. Стандартная и каноническая формы. Существование оптимального решения (без доказательства). Лемма Фаркаша.

**Общая постановка**
Задача линейного программирования (ЛП) — это оптимизация линейной функции при линейных ограничениях.
$$\max_{x \in \mathbb{R}^n} c^T x \quad \text{при} \quad Ax \le b$$

**Формы записи**
*   **Стандартная форма**: $\max c^T x$, $Ax \le b$, $x \ge 0$.
*   **Каноническая форма**: $\min c^T x$, $Ax = b$, $x \ge 0$.
Эти формы эквивалентны. Любую задачу ЛП можно к ним привести:
*   $\min$ заменить на $\max$ со знаком минус.
*   $a^T x \ge b$ заменить на $-a^T x \le -b$.
*   $a^T x = b$ заменить на $a^T x \le b$ и $-a^T x \le -b$.
*   Свободную переменную $x_j$ заменить на $x_j^+ - x_j^-$, где $x_j^+, x_j^- \ge 0$.
*   Неравенство $a^T x \le b$ заменить на $a^T x + s = b$, $s \ge 0$ (добавочная переменная).

**Допустимые и оптимальные решения**
*   **Множество допустимых решений** $P = \{x \in \mathbb{R}^n \mid Ax \le b\}$ — это выпуклый многогранник (пересечение полупространств).
*   **Оптимальное решение** $x^* \in \arg\max_{x \in P} c^T x$.
*   **Существование**: Если задача ЛП ограничена (т.е. $P \ne \emptyset$ и целевая функция ограничена сверху на $P$) и $P$ замкнуто (что верно для $Ax \le b$), то по теореме Вейерштрасса оптимальное решение существует.

**Лемма Фаркаша (Теорема 6.2)**
Для любых $A \in \mathbb{R}^{m \times n}$ и $c \in \mathbb{R}^n$ выполняется ровно одно из двух условий:
1.  $\exists y \ge 0: A^T y = c$ (т.е. $c$ принадлежит конусу, натянутому на строки $A$).
2.  $\exists z \in \mathbb{R}^n: Az \le 0$ и $c^T z > 0$ (т.е. существует гиперплоскость, отделяющая $c$ от конуса).
*   *Геометрический смысл*: Либо вектор $c$ является неотрицательной комбинацией строк $A$, либо существует вектор $z$, образующий тупой угол со всеми строками $A$ ($a_i^T z \le 0$) и острый угол с $c$ ($c^T z > 0$).
*   *Доказательство*: Если (1) не выполнено, то $c \notin K = \{A^T y \mid y \ge 0\}$. Берется ближайшая к $c$ точка $v \in K$ и полагается $z = c - v$. Затем доказывается, что $Az \le 0$ (иначе можно найти точку на отрезке $[a_k, v]$, лежащую ближе к $c$) и $c^T z > 0$.

### 8. Теорема о двойственности в линейном программировании (без доказательства леммы Фаркаша). Системы линейных неравенств. Полиномиальные алгоритмы решения лин. прог. Общая идея алгоритма внутренней точки. Преобразование задачи для нахождения какой-то внутренней точки.

**Теорема о двойственности (Теорема 6.3)**
Рассмотрим пару задач (прямая и двойственная):
*   **Прямая (P)**: $\max c^T x$, $Ax \le b$.
*   **Двойственная (D)**: $\min b^T y$, $A^T y = c$, $y \ge 0$.

**Утверждения теоремы:**
1.  **Слабая двойственность**: Для любых допустимых $x$ и $y$ выполняется $c^T x \le b^T y$.
2.  **Сильная двойственность**: Если одна из задач имеет оптимальное решение, то и другая имеет оптимальное решение, и значения целевых функций на оптимумах равны: $c^T x^* = b^T y^*$.
3.  Если одна из задач неограничена, то другая не имеет допустимых решений.
*   *Ключевой шаг доказательства*: Из оптимальности $x^*$ следует, что не существует $z$ с $A_0 z \le 0$ и $c^T z > 0$ (иначе $x^* + \epsilon z$ был бы лучше). По лемме Фаркаша отсюда $\exists y_0 \ge 0: A_0^T y_0 = c$. Дополняя $y_0$ нулями для неактивных ограничений, получаем допустимое решение $y$ двойственной задачи, для которого $b^T y = c^T x^*$.

**Сведение к системе неравенств**
Оптимальное решение $(x^*, y^*)$ пары (P)-(D) является решением системы линейных неравенств:
$$Ax \le b,\quad A^T y = c,\quad y \ge 0,\quad b^T y - c^T x \le 0$$
По слабой двойственности $b^T y - c^T x \ge 0$, поэтому добавленное неравенство эквивалентно $b^T y = c^T x$. Таким образом, решение ЛП сводится к нахождению решения системы линейных неравенств.

**Полиномиальные алгоритмы**
*   **Длина входа**: $L = \Theta(mnw)$, где $w$ — число битов для кодирования чисел. Полиномиальный алгоритм должен работать за $O(L^{O(1)})$.
*   **Симплекс-метод**: Работает хорошо на практике, но в худшем случае имеет экспоненциальное время.
*   **Метод эллипсоидов**: Первый полиномиальный алгоритм, но медленный на практике.
*   **Метод внутренней точки (например, алгоритм Кармаркара)**: Полиномиальный и эффективный на практике.

**Идея метода внутренней точки**
Алгоритм движется внутри допустимого многогранника $Ax < b$ к оптимальному решению, используя штрафную ("барьерную") функцию.
*   **Барьерная функция**: $g(x) = \sum_{i=1}^m \ln(b_i - a_i^T x)$. Она стремится к $-\infty$ при приближении к границе ($b_i - a_i^T x \to 0$).
*   **Барьерная задача**: $\max (w c^T x + g(x))$ при $Ax \le b$. Для каждого $w>0$ решение $x_w^*$ лежит на **центральном пути**. При $w \to \infty$ влияние барьера ослабевает, и $x_w^*$ стремится к решению исходной задачи.
*   **Алгоритм**: Идет по центральному пути, увеличивая $w$ и делая шаг Ньютона для приближения к $x_{w_{k+1}}^*$ из предыдущей точки $x_{w_k}^*$.

**Как найти внутреннюю точку**
Для поиска начальной внутренней точки $x_0$ ($Ax_0 < b$) решается вспомогательная задача:
$$\max t \quad \text{при} \quad Ax + t \mathbf{1} \le b$$
Если оптимальное значение $t^* > 0$, то соответствующая точка $x^*$ является внутренней. У этой задачи всегда есть очевидная внутренняя точка, например, $x=0$, $t = -\max_i|b_i|-1$.

### 9. Задача о максимальном потоке в сети. Формулировка в лин. прог. Дополняющие пути. Алгоритм Форда–Фалкерсона. Анализ его трудоемкости для целочисленной задачи о максимальном потоке. Теорема о максимальном потоке и минимальном разрезе.

**Определения**
*   **Транспортная сеть**: Орграф $G = (V, E, c)$, где $c: E \to \mathbb{R}_{\ge 0}$ — пропускные способности, $s$ — исток ($\deg^-(s)=0$), $t$ — сток ($\deg^+(t)=0$).
*   **Поток**: Функция $f: E \to \mathbb{R}_{\ge 0}$, удовлетворяющая:
    1.  $0 \le f(u,v) \le c(u,v)$ (ограничение пропускной способности).
    2.  Закон сохранения: для любой $v \in V \setminus \{s, t\}$, $\sum_{(u,v)\in E} f(u,v) = \sum_{(v,w)\in E} f(v,w)$.
*   **Величина потока**: $|f| = \sum_{(s,v)\in E} f(s,v) = \sum_{(u,t)\in E} f(u,t)$.

**Формулировка в ЛП**
Задача о максимальном потоке — это задача ЛП:
$$\max \sum_{(s,v)\in E} f(s,v)$$
при ограничениях: $0 \le f(u,v) \le c(u,v)$, $\forall (u,v) \in E$, и законе сохранения для всех $v \in V\setminus\{s,t\}$.

**Остаточная сеть и дополняющие пути**
*   **Остаточная сеть** $G_f = (V, E_f, c_f)$:
    *   Прямое ребро $(u,v) \in E_f$ с $c_f(u,v) = c(u,v) - f(u,v)$, если $c(u,v) > f(u,v)$.
    *   Обратное ребро $(v,u) \in E_f$ с $c_f(v,u) = f(u,v)$, если $f(u,v) > 0$.
*   **Дополняющий путь**: Путь из $s$ в $t$ в $G_f$. Его пропускная способность $\delta(P) = \min c_f(e)$ на пути. Увеличив поток на $\delta$ вдоль этого пути (добавляя на прямых ребрах и вычитая на обратных), получим новый поток $f'$ с $|f'| = |f| + \delta$.

**Алгоритм Форда–Фалкерсона**
1.  Начать с нулевого потока $f \equiv 0$.
2.  Пока существует дополняющий путь $P$ из $s$ в $t$ в $G_f$:
    *   Найти $P$ (например, BFS/DFS).
    *   $\delta = \min_{e \in P} c_f(e)$.
    *   Увеличить поток вдоль $P$ на $\delta$.
3.  Вернуть $f$.

**Анализ для целочисленных пропускных способностей**
Если все $c(e)$ — целые числа, то на каждой итерации $\delta \ge 1$ (целое). Так как величина потока ограничена сверху, например, суммой $c(s, v)$, алгоритм завершится не более чем за $|f_{\max}|$ итераций. Каждая итерация (поиск пути) может быть выполнена за $O(|E|)$ (BFS/DFS). Следовательно, общее время работы для целочисленных сетей: $O(|E| \cdot |f_{\max}|)$.

**Теорема о максимальном потоке и минимальном разрезе (Теорема 8.1)**
Для потока $f$ в транспортной сети эквивалентны:
1.  $f$ — максимальный поток.
2.  В остаточной сети $G_f$ нет дополняющего пути из $s$ в $t$.
3.  Существует $s$-$t$-разрез $(S, T)$ (разбиение $V = S \cup T$, $s \in S$, $t \in T$) такой, что $|f| = c(S, T) = \sum_{u\in S, v\in T} c(u, v)$.

**Следствие**: Величина максимального потока равна пропускной способности минимального разреза.

### 10. Приближённые алгоритмы. Задача о вершинном покрытии минимальной мощности, приближённый полиномиальный алгоритм для её решения. Задача о вершинном покрытии минимальной стоимости, её приближённое решение при помощи лин. прог.

**Приближённый алгоритм**
Алгоритм $A$ называется $\rho$-приближением ($\rho \ge 1$), если для любого входа $x$:
*   Для задачи **максимизации**: $A(x) \ge \frac{OPT(x)}{\rho}$.
*   Для задачи **минимизации**: $A(x) \le OPT(x) \cdot \rho$.

**Задача о вершинном покрытии (минимальной мощности)**
*   **Вход**: Неориентированный граф $G = (V, E)$.
*   **Цель**: Найти $C \subseteq V$ минимального размера, такое что каждое ребро $e \in E$ инцидентно хотя бы одной вершине из $C$.
*   **2-приближенный алгоритм (жадный по паросочетанию)**:
    1.  $C = \emptyset$.
    2.  Пока $E \ne \emptyset$:
        *   Выбрать произвольное ребро $\{u, v\} \in E$.
        *   Добавить $u$ и $v$ в $C$ ($C := C \cup \{u, v\}$).
        *   Удалить из $E$ все ребра, инцидентные $u$ или $v$.
    3.  Вернуть $C$.
*   **Доказательство** ($|C| \le 2 \cdot OPT$):
    *   Пусть $M$ — множество выбранных на шаге 2 ребер. $|C| = 2|M|$.
    *   Любое вершинное покрытие $C^*$ должно содержать хотя бы по одной вершине из каждого ребра $M$, и эти вершины различны (так как ребра в $M$ не смежны). Следовательно, $|M| \le |C^*|$.
    *   Таким образом, $|C| = 2|M| \le 2|C^*| = 2 \cdot OPT$.

**Задача о вершинном покрытии (минимальной стоимости)**
*   **Вход**: Граф $G = (V, E)$ и веса вершин $w(v) > 0$.
*   **Цель**: Найти $C \subseteq V$ минимального суммарного веса $\sum_{v \in C} w(v)$, покрывающее все ребра.
*   **Целочисленное программирование**:
    *   Переменная $x_v \in \{0, 1\}$ (1, если $v \in C$).
    *   Цель: $\min \sum w(v) x_v$.
    *   Ограничения: $x_u + x_v \ge 1$ для каждого ребра $\{u, v\} \in E$.
*   **2-приближенный алгоритм через LP-релаксацию**:
    1.  Решить **LP-релаксацию** (ослабить условие $x_v \in \{0,1\}$ до $0 \le x_v \le 1$), получив оптимальное дробное решение $\{x_v^*\}$.
    2.  Округлить: $C = \{v \mid x_v^* \ge \frac12\}$.
*   **Доказательство** ($\sum_{v \in C} w(v) \le 2 \cdot OPT$):
    *   **Корректность**: Для любого ребра $\{u, v\}$: $x_u^* + x_v^* \ge 1 \implies$ хотя бы одна из переменных $\ge \frac12 \implies$ хотя бы одна вершина попала в $C$.
    *   **Аппроксимация**: Для $v \in C$, $x_v^* \ge 1/2 \implies 1 \le 2x_v^*$.
        $$\sum_{v \in C} w(v) \le \sum_{v \in V} w(v) \cdot (2x_v^*) = 2 \sum_{v \in V} w(v) x_v^* \le 2 \sum_{v \in V} w(v) \hat{x}_v = 2 \cdot OPT,$$
        где $\hat{x}_v$ — оптимальное целочисленное решение, а $\sum w(v) x_v^*$ — не более него, так как $\hat{x}_v$ допустимо для LP.

### 11. Метрическая задача коммивояжёра. Алгоритм двойного дерева и алгоритм Кристофидеса–Сердюкова для её приближённого решения.

**Метрическая TSP**
*   **Вход**: Полный граф $K_n$ с весами $c: E \to \mathbb{R}_{\ge 0}$, удовлетворяющими неравенству треугольника: $\forall u, v, w: c(u, v) + c(v, w) \ge c(u, w)$.
*   **Цель**: Найти гамильтонов цикл минимального веса.
*   В общем случае TSP неаппроксимируема, но для метрической версии существуют алгоритмы с константной аппроксимацией.

**Алгоритм двойного дерева (2-приближение)**
1.  Найти минимальное остовное дерево (MST) $T$ графа $G$.
2.  Создать мультиграф $T'$, продублировав все ребра $T$ (все степени станут четными).
3.  Найти в $T'$ Эйлеров цикл $u_1 \to u_2 \to \dots \to u_k \to u_1$.
4.  Построить гамильтонов цикл $H$, проходя по Эйлерову циклу и пропуская уже посещенные вершины ("срезая" их).
*   **Оценка**: $c(H) \le c(\text{эйлерова цикла}) = 2c(T) \le 2c(H^*)$, где $H^*$ — оптимальный гамильтонов цикл (так как $T$ — подграф $H^*$ без одного ребра).
*   **Аппроксимация**: 2.

**Алгоритм Кристофидеса–Сердюкова (3/2-приближение)**
1.  Найти минимальное остовное дерево $T$ графа $G$.
2.  Пусть $V_0 \subseteq V$ — множество вершин с нечетной степенью в $T$ (их четное число по лемме о рукопожатиях).
3.  Найти в $G$ минимальное по весу совершенное паросочетание $M$ на вершинах $V_0$.
4.  Создать мультиграф $T' = T \cup M$ (все вершины имеют четную степень).
5.  Найти в $T'$ Эйлеров цикл.
6.  Построить гамильтонов цикл $H$, срезая повторные вершины.
*   **Оценка**: $c(H) \le c(T) + c(M)$.
    *   $c(T) \le c(H^*)$ (как и в алгоритме двойного дерева).
    *   $c(M) \le \frac12 c(H^*)$. (Доказательство: возьмем вершины $V_0$ в порядке их появления в $H^*$ и рассмотрим два паросочетания: $M_1$ = (1-2, 3-4, ...) и $M_2$ = (2-3, 4-5, ...). По неравенству треугольника $c(M_1)+c(M_2) \le c(H^*)$, значит, $\min(c(M_1), c(M_2)) \le \frac12 c(H^*)$. А $M$ — минимальное, поэтому $c(M) \le \frac12 c(H^*)$.)
*   **Аппроксимация**: $1 + \frac12 = \frac32$.

### 12. Вероятностные алгоритмы. Задача проверки умножения матриц. Алгоритм Фрейвальдса. Задача проверки равенства многочленов. Лемма Шварца–Зиппеля.

**Вероятностные алгоритмы**
*   **Лас-Вегас**: Всегда выдает правильный ответ, время работы — случайная величина.
*   **Монте-Карло**: Может ошибаться с заданной вероятностью.
    *   **С односторонней ошибкой**: Если ответ "нет", то он всегда верен. Ошибка только при ответе "да".
    *   **С двусторонней ошибкой**: Ошибка возможна при любом ответе.

**Алгоритм Фрейвальдса (проверка $AB = C$)**
*   **Задача**: Даны $A, B, C \in \mathbb{R}^{n \times n}$. Верно ли, что $AB = C$?
*   **Алгоритм (Монте-Карло)**:
    1.  Выбрать случайный вектор $\mathbf{r} \in \{0,1\}^n$ (равновероятно, покоординатно).
    2.  Вычислить $A(B\mathbf{r})$ и $C\mathbf{r}$.
    3.  Если $A(B\mathbf{r}) = C\mathbf{r}$, вернуть "Да", иначе "Нет".
*   **Анализ**:
    *   Если $AB = C$, то $A(Br) = Cr$ всегда $\implies$ ответ всегда "Да".
    *   Если $AB \ne C$, то существует строка $i$, где $(AB)_{i,*} \ne C_{i,*}$. Для фиксированных $r_j, j \ne k$, уравнение $(AB-C)_{i,*} r = 0$ имеет не более одного решения для $r_k \in \{0,1\}$. Следовательно, $\Pr[\text{ошибка}] \le \frac12$.
*   **Уменьшение ошибки**: Повторить $k$ раз. Если хотя бы один раз ответ "Нет" — вернуть "Нет". Вероятность ошибки $\le (1/2)^k$.

**Проверка равенства многочленов (PIT)**
*   **Задача**: Даны два многочлена $p(x_1, \dots, x_n)$ и $q(x_1, \dots, x_n)$ (например, заданные схемой). Проверить, что $p \equiv q$. Или что $f = p - q \equiv 0$.
*   **Лемма Шварца–Зиппеля (Лемма 10.1)**: Пусть $f(x_1, \dots, x_n) \not\equiv 0$ — ненулевой многочлен над полем $F$ степени $d$. Пусть $S$ — конечное подмножество $F$. Если $r_1, \dots, r_n$ выбраны независимо и равномерно случайно из $S$, то $$\Pr[f(r_1, \dots, r_n) = 0] \le \frac{d}{|S|}.$$
*   **Доказательство**: Индукция по $n$. База $n=1$ — у многочлена не более $d$ корней. Шаг: $f = \sum_{i=0}^d p_i(x_1, \dots, x_{n-1}) x_n^i$. Пусть $i_{\max}$ — максимальный индекс, при котором $p_{i_{\max}} \not\equiv 0$. Тогда $\Pr[f=0] \le \Pr[p_{i_{\max}}=0] + \Pr[f(r_1, \dots, r_{n-1}, x_n)=0 \mid p_{i_{\max}} \ne 0] \le \frac{d - i_{\max}}{|S|} + \frac{i_{\max}}{|S|} = \frac{d}{|S|}$.
*   **Алгоритм (Монте-Карло)**: Выбрать $r_1, \dots, r_n$ случайно из $S$ (например, из $\{1, \dots, 2d\}$). Вычислить $f(r_1, \dots, r_n)$. Если $f \ne 0$, то $f \not\equiv 0$. Если $f = 0$, то с вероятностью $\ge 1 - \frac{d}{|S|}$ многочлен тождественно равен нулю.

### 13. Хеш таблицы, ключи. Амортизированное и ожидаемое время работы. Время работы хеш таблицы с открытой адресацией при идеализированном случайном равновероятном хешировании.

**Хеш-таблица с открытой адресацией**
*   **Структура**: Массив $T$ размера $m$. Храним $n$ ключей, $n \le m$. Используется хеш-функция $h: U \to \{0, \dots, m-1\}$.
*   **Коллизия**: $h(x) = h(y)$ для $x \ne y$. Разрешается линейным поиском (линейное зондирование).
*   **Операции**:
    *   **Вставка**: Ищем пустую ячейку ($\emptyset$) или "могилку" ($\dagger$), начиная с $h(x)$. Вставляем. Если $n$ превышает порог (например, $2m$), увеличиваем размер таблицы и перехешируем.
    *   **Поиск**: Идем с $h(x)$, пока не найдем $x$ или $\emptyset$ (тогда $x$ нет).
    *   **Удаление**: Помечаем ячейку как "могилка" ($\dagger$), не удаляя элемент физически. (Нужно, чтобы поиск не остановился на месте удаленного элемента).

**Идеализированное предположение**
Хеш-функция $h$ выбрана случайно и равновероятно из множества всех функций $U \to \{0, \dots, m-1\}$. Это значит, что для каждого ключа ячейка выбирается независимо и равномерно.
*(На практике используют универсальные семейства, см. билет 14).*

**Ожидаемое время работы (без учета перестроек)**
Пусть $m = 2n$ (коэффициент заполнения $\alpha = n/m = 1/2$). Ожидаемое время поиска/вставки $O(1)$.
*   **Доказательство**: Ожидаемое время пропорционально длине "хвоста" серии занятых ячеек от $t = h(x)$. Пусть $p_k$ — вероятность того, что первые $k$ ячеек $t, t+1, \dots, t+k-1$ заняты. Можно показать, что $q_\ell$ (вероятность серии длины $\ell$) экспоненциально мала: $q_\ell \le e^{-\Omega(\ell)}$.
*   $\mathbb{E}[\text{время}] = O\left(\sum_{\ell=0}^n q_\ell \cdot \ell^2\right)$. Так как $q_\ell$ убывает экспоненциально, эта сумма сходится к $O(1)$.

**Амортизированное время работы**
Учитывает перестройку таблицы (resize). При вставке, когда $n$ достигает $m/2$, размер таблицы увеличивается вдвое (до $2m$), и все $n$ элементов перехешируются. Стоимость перестройки — $\Theta(n)$. Такую операцию можно "оплатить" за счет предыдущих вставок (каждая вставка резервирует константное время на будущую перестройку). Следовательно, **амортизированное** ожидаемое время всех операций — $O(1)$.

### 14. Универсальные семейства хеш-функций. 2-универсальность семейства multiply-shift. Хеш таблицы с разрешением коллизий цепочками.

**Универсальное семейство**
Семейство $H$ хеш-функций $h: U \to \{0, \dots, m-1\}$ называется **универсальным**, если для любых различных $x, y \in U$:
$$\Pr_{h \in H}[h(x) = h(y)] \le \frac{1}{m}.$$
Это позволяет анализировать поведение хеш-таблиц в среднем, не предполагая, что входные данные случайны.

**Пример: multiply-add**
Пусть $U = \{0, \dots, p-1\}$, $p$ — простое, $p \ge |U|$. Семейство $H = \{h_{a,b} \mid a \in \{1, \dots, p-1\}, b \in \{0, \dots, p-1\}\}$, где
$$h_{a,b}(x) = ((ax + b) \bmod p) \bmod m$$
является универсальным.

**Пример: multiply-shift (2-универсальное)**
*   Пусть ключи — $w$-битовые числа: $U = \{0, \dots, 2^w-1\}$. Размер таблицы $m = 2^\ell$.
*   Выберем нечетное $w$-битовое число $a$ (случайно). Определим:
    $$h_a(x) = (a \cdot x \ \gg (w - \ell)) \ \& \ (2^\ell - 1)$$
    Т.е. берем произведение $a \cdot x$ ( $2w$ бит), отбрасываем $w-\ell$ младших битов и берем младшие $\ell$ бит из оставшихся.
*   **Свойство**: Семейство $\{h_a\}$ является **2-универсальным**, т.е. $\Pr[h_a(x) = h_a(y)] \le 2/m$.

**Хеш-таблица с цепочками**
*   **Структура**: Массив $T$ размера $m$, где каждая ячейка $T[i]$ — это указатель на голову списка.
*   **Операции**: $h(x)$ определяет список $T[h(x)]$, в который вставляется, из которого удаляется или в котором ищется элемент $x$.
*   **Анализ**:
    *   Пусть $n$ — количество элементов. Коэффициент заполнения $\alpha = n/m$.
    *   Ожидаемая длина списка для случайного $h$ из универсального семейства: $\mathbb{E}[\text{длина}] \le 1 + \alpha$ (поиск своего элемента). Для вставки/удаления — $O(1)$.
    *   **Доказательство**: Для фиксированного $x$, индикаторная переменная $I_y = [h(y) = h(x)]$. $\mathbb{E}[\sum_{y \ne x} I_y] \le (n-1) \cdot \frac{1}{m} \le \alpha$. Добавляем сам $x$, получаем $\le 1+\alpha$.

### 15. Совершенное хеширование. Двухуровневое совершенное хеширование. (На основе конспекта Охотина)

**Совершенное хеширование (Static)**
*   **Задача**: Дан статический набор ключей $S \subseteq U$, $|S| = N$. Нужно построить структуру данных для поиска ключа за $O(1)$ времени в худшем случае, используя $O(N)$ памяти.
*   **Идея**: Двухуровневая схема Фредмана, Комлоша и Семереди (1984).
    1.  **Первый уровень**: Выбираем хеш-функцию $h$ из универсального семейства, отображающую $U$ в $\{0, \dots, N-1\}$. Для каждого $i$, пусть $n_i = |\{x \in S \mid h(x) = i\}|$.
    2.  **Второй уровень**: Для каждого $i$ строим **идеальную** (без коллизий) хеш-таблицу размера $m_i = n_i^2$, используя собственную хеш-функцию $h_i$ из универсального семейства. Так как $m_i = n_i^2$, вероятность отсутствия коллизий при случайном выборе $h_i$ $\ge 1/2$.
*   **Анализ памяти**:
    *   Ожидаемый суммарный размер вторичных таблиц:
        $\mathbb{E}[\sum_i n_i^2] = N + 2 \mathbb{E}[\sum_i \binom{n_i}{2}] = N + 2 \binom{N}{2} \cdot \frac{1}{N} = 2N - 1 = O(N)$.
        (Первое равенство: $n_i^2 = n_i + 2\binom{n_i}{2}$. Второе: $\sum_i \binom{n_i}{2}$ — количество коллидирующих пар, матожидание одной пары $\le 1/N$.)
    *   С вероятностью $\ge 1/2$ суммарный размер $\le 4N$.
*   **Поиск**: Вычисляем $i = h(x)$, затем $j = h_i(x)$ и проверяем ячейку $T_i[j]$. Время $O(1)$ в худшем случае (нет коллизий во вторичных таблицах).

### 16. Задача проверки простоты числа. Тест Ферма. Числа Кармайкла. Алгоритм Миллера–Рабина. (На основе конспекта Охотина)

**Тест Ферма (вероятностный)**
*   Основан на малой теореме Ферма: если $n$ простое, то $a^{n-1} \equiv 1 \pmod n$ для всех $a$, не кратных $n$.
*   **Алгоритм**: Выбрать случайное $a \in \{2, \dots, n-2\}$. Если $a^{n-1} \not\equiv 1 \pmod n$, вернуть "составное". Иначе — "возможно, простое".
*   **Проблема**: Существуют **числа Кармайкла** (например, 561 = 3 * 11 * 17). Это составные числа, для которых $a^{n-1} \equiv 1 \pmod n$ для всех $a$, взаимно простых с $n$. Тест Ферма почти всегда ошибается на таких числах.

**Алгоритм Миллера–Рабина**
Проверяет не только условие Ферма, но и отсутствие нетривиальных квадратных корней из 1 по модулю простого числа.
*   **Идея**: Для простого $p$, если $x^2 \equiv 1 \pmod p$, то $x \equiv \pm 1 \pmod p$.
*   **Алгоритм**:
    1.  Записать $n-1 = 2^t \cdot u$, где $u$ нечетно.
    2.  Выбрать случайное $a \in \{2, \dots, n-1\}$.
    3.  Вычислить $x_0 = a^u \mod n$.
    4.  Последовательно возводить в квадрат: $x_{i+1} = x_i^2 \mod n$ для $i=0, \dots, t-1$.
    5.  Если $x_t \not\equiv 1 \pmod n$, вернуть "составное" (не прошло условие Ферма).
    6.  Найти первый $i$, для которого $x_i \equiv 1 \pmod n$. Если $x_{i-1} \not\equiv \pm 1 \pmod n$, вернуть "составное" (найден нетривиальный корень из 1).
    7.  Если все проверки пройдены, вернуть "возможно, простое".
*   **Свойства**:
    *   Для составного $n$ вероятность ошибочно объявить его "возможно, простым" $\le 1/4$.
    *   Повторяя тест $k$ раз с разными $a$, вероятность ошибки $\le (1/4)^k$.
    *   Алгоритм является **Монте-Карло с односторонней ошибкой**: ответ "составное" всегда верен, ответ "простое" может быть ошибочным с малой вероятностью.
*   **Время работы**: $O(k \cdot (\log n)^3)$.

### 17. Онлайн алгоритмы. Задача кэширования. Помечающий алгоритм. (На основе конспекта Охотина)

**Онлайн алгоритмы**
Обрабатывают входные данные по мере их поступления, не зная будущего. Принимают решения на основе уже полученной информации. Качество оценивается **конкурентным анализом**: коэффициент конкурентноспособности $\rho$ означает, что для любой входной последовательности стоимость алгоритма $\le \rho \cdot \text{OPT}$, где OPT — стоимость оптимального офлайн алгоритма, знающего всю последовательность.

**Задача кэширования (paging)**
*   **Вход**: Последовательность обращений к страницам памяти. Кэш может хранить $k$ страниц. При обращении к странице:
    *   Если страница в кэше — "попадание" (hit), стоимость 0.
    *   Если страницы нет — "промах" (miss), нужно загрузить ее в кэш, возможно, выгрузив другую. Стоимость 1.
*   **Цель**: Минимизировать количество промахов.

**Помечающий алгоритм (marking algorithm)**
*   **Принцип**: Страницы в кэше могут быть "помечены" (защищены от вытеснения). Алгоритм разбит на **фазы**.
*   **Алгоритм**:
    1.  В начале фазы все страницы в кэше не помечены.
    2.  При обращении к странице $p$:
        *   Если $p$ в кэше, пометить ее.
        *   Если $p$ нет в кэше:
            *   Если есть непомеченные страницы, выбрать случайную непомеченную страницу, выгрузить ее, загрузить $p$ и пометить $p$.
            *   Если все страницы помечены, начать новую фазу: снять все пометки и обработать запрос как промах в новой фазе.
*   **Свойства**:
    *   В каждой фазе загружается не более $k$ различных страниц.
    *   В каждой фазе число промахов равно числу различных страниц, запрошенных в этой фазе, но отсутствовавших в кэше в начале фазы.
*   **Анализ** (Фиат и др., 1991):
    *   Для любого детерминированного офлайн алгоритма, число промахов $\ge$ (количество различных страниц в фазе) / 2.
    *   Ожидаемое число промахов помечающего алгоритма $\le$ (количество различных страниц в фазе) $\cdot H_k$, где $H_k = 1 + 1/2 + \dots + 1/k$ — гармоническое число.
    *   Следовательно, помечающий алгоритм является $2H_k$-конкурентным. $\lim_{k \to \infty} 2H_k / \ln k = 2$, т.е. он в $O(\log k)$ раз хуже оптимального.

### 18. Задача нахождения минимального остовного дерева. Алгоритмы Ярника–Прима–Дейкстры и Краскала. Системы непересекающихся множеств (Union-Find). (На основе конспектов Косолобова и Охотина)

**Задача**
Дан связный неориентированный граф $G = (V, E)$ с весами $w: E \to \mathbb{R}$. Найти остовное дерево $T \subseteq E$ минимального суммарного веса $\sum_{e \in T} w(e)$.

**Алгоритм Прима (Ярника–Прима–Дейкстры)**
*   **Идея**: Растет "дерево" от одной вершины, на каждом шаге добавляя самое легкое ребро, соединяющее дерево с остальными вершинами.
*   **Структура данных**: Очередь с приоритетами для вершин, не вошедших в дерево. Ключ $d[v]$ — минимальный вес ребра из $v$ в текущее дерево, $\pi[v]$ — вершина дерева, соединяющая $v$ с ним.
*   **Алгоритм**:
    1.  Выбрать стартовую вершину $s$. $d[s] = 0$, $d[v] = \infty$ для $v \ne s$.
    2.  Пока очередь не пуста:
        *   Извлечь $u$ с минимальным $d[u]$, добавить ребро $(u, \pi[u])$ в $T$.
        *   Для каждого соседа $v$ вершины $u$:
            *   Если $v$ в очереди и $w(u,v) < d[v]$, обновить $d[v] = w(u,v)$ и $\pi[v] = u$.
*   **Время работы**: $O(|E| \log |V|)$ (с использованием двоичной кучи).

**Алгоритм Краскала**
*   **Идея**: Добавлять ребра в порядке возрастания веса, если они соединяют разные компоненты связности текущего леса.
*   **Структура данных**: Система непересекающихся множеств (Union-Find) для хранения компонент связности.
*   **Алгоритм**:
    1.  Отсортировать ребра по весу.
    2.  Инициализировать Union-Find: каждая вершина в своем множестве.
    3.  Для каждого ребра $(u, v)$ в порядке возрастания веса:
        *   Если $\text{find}(u) \ne \text{find}(v)$, добавить $(u, v)$ в $T$ и выполнить $\text{union}(u, v)$.
*   **Время работы**: $O(|E| \log |E|) = O(|E| \log |V|)$ (доминирует сортировка).

**Система непересекающихся множеств (Union-Find)**
*   **Операции**:
    *   `make_set(x)`: создать множество из элемента $x$.
    *   `find(x)`: вернуть идентификатор (представителя) множества, содержащего $x$.
    *   `union(x, y)`: объединить множества, содержащие $x$ и $y$.
*   **Эффективная реализация (Лес непересекающихся множеств)**:
    *   **Деревья**: Каждое множество — это дерево. Корень — представитель.
    *   **Эвристики**:
        1.  **Объединение по рангу**: При `union` подвешивать дерево с меньшим рангом (высотой) к корню дерева с большим рангом.
        2.  **Сжатие пути**: При `find` перенаправлять все вершины на пути непосредственно к корню.
*   **Время работы**: Почти линейное. Если выполнить $m$ операций, из которых $n$ — `make_set`, то время $O(m \cdot \alpha(n))$, где $\alpha(n)$ — обратная функция Аккермана (растет крайне медленно, $\le 4$ для практических $n$).

### 19. Задача булевой выполнимости (SAT). Задачи kSAT и MAX-kSAT. Вероятностный и детерминированный приближённые алгоритмы для MAX-3SAT. (На основе конспекта Охотина)

**Задача SAT**
*   **Задача**: Дана булева формула в КНФ (конъюнкция дизъюнкций). Существует ли набор переменных, делающий формулу истинной?
*   **$k$SAT**: Ограничение, что каждый дизъюнкт (clause) содержит не более $k$ литералов. $k$SAT NP-полна для $k \ge 3$.
*   **MAX-$k$SAT**: Оптимизационная задача. Найти набор переменных, максимизирующий количество **выполненных** (истинных) дизъюнктов.

**Вероятностный алгоритм для MAX-3SAT**
*   **Идея**: Выбрать значения всех переменных случайно и независимо (0 или 1 с вероятностью 1/2).
*   **Анализ**:
    *   Вероятность того, что конкретный дизъюнкт из 3 литералов выполнен, равна $1 - (1/2)^3 = 7/8$.
    *   По линейности матожидания, $\mathbb{E}[\# \text{выполненных дизъюнктов}] = \frac{7}{8}m$.
    *   Следовательно, существует набор, на котором выполнено не менее $7/8$ дизъюнктов.
    *   Этот алгоритм является $8/7$-приближением (так как максимальное число выполненных дизъюнктов $\le m$). $\frac{8}{7} \cdot \frac{7}{8}m = m$.
*   **Вероятность успеха**: Можно показать, что $\Pr[\# \text{выполненных} \ge 7m/8] \ge \frac{1}{8m}$. Повторяя алгоритм $O(m)$ раз, можно найти хороший набор с высокой вероятностью.

**Детерминированный алгоритм (дерандомизация)**
*   **Метод**: Условные математические ожидания.
*   **Идея**: Переменные назначаются последовательно. На каждом шаге вычисляется матожидание числа выполненных дизъюнктов при условии уже сделанных назначений. Выбирается то значение (0 или 1), которое дает большее матожидание. Так как изначально матожидание было $7m/8$, а в конце оно становится равным фактическому числу выполненных дизъюнктов, алгоритм гарантированно найдет набор с $\ge 7m/8$ истинными дизъюнктами.
*   **Вычисление матождиания**: Для невыполненного и неопределенного дизъюнкта из $t$ неопределенных переменных, вероятность его выполнения равна $1 - (1/2)^t$. Это можно эффективно подсчитать, присвоив вес дизъюнкту.
*   **Результат**: Существует детерминированный полиномиальный $8/7$-приближенный алгоритм для MAX-3SAT.

**Результат о неаппроксимируемости**
Хостад (2001) доказал, что существование $(8/7 - \epsilon)$-приближенного алгоритма для MAX-3SAT эквивалентно $P = NP$.
